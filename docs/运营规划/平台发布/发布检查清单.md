## 一、检查清单使用说明

### 1.1 使用时机

| 阶段 | 使用场景 |
|------|----------|
| 内容完成后 | CEO自查内容质量 |
| 交付用户前 | 确保内容可发布 |
| 发布操作时 | 用户按清单操作 |
| 发布完成后 | 验证发布效果 |

### 1.2 检查项状态

| 标记 | 含义 |
|------|------|
| ☐ | 待检查 |
| ☑ | 已通过 |
| ⚠ | 有问题需修改 |
| N/A | 不适用 |

## 二、内容格式检查

### 2.1 标题检查

| 序号 | 检查项 | 标准 | 状态 |
|------|--------|------|------|
| 1 | 标题长度 | 20-35字为宜 | ☐ |
| 2 | 关键词位置 | 核心关键词在前20字 | ☐ |
| 3 | 时效性 | 包含年份（如2026） | ☐ |
| 4 | 吸引力 | 包含数字/效果词 | ☐ |
| 5 | 无敏感词 | 不含违规词汇 | ☐ |
| 6 | 无夸大 | 避免"震惊""必看"等 | ☐ |

**标题模板参考**：
```
✅ 一文讲清XXX（2026最新版）
✅ XXX是什么？一篇文章讲清楚
✅ 5个XXX技巧，让你的XXX提升300%
❌ 震惊！这个XXX你一定要看
❌ 不看后悔系列...
```

### 2.2 正文检查

| 序号 | 检查项 | 标准 | 状态 |
|------|--------|------|------|
| 1 | 字数达标 | 1500-3000字 | ☐ |
| 2 | 段落清晰 | 每段3-5行 | ☐ |
| 3 | 标题层级 | 使用##/### | ☐ |
| 4 | 重点标注 | **关键内容加粗** | ☐ |
| 5 | 开头吸引 | 首段说明价值 | ☐ |
| 6 | 结尾引导 | 引导点赞/关注 | ☐ |
| 7 | 逻辑连贯 | 段落过渡自然 | ☐ |
| 8 | 错别字 | 无明显错误 | ☐ |

### 2.3 代码块检查（技术内容）

| 序号 | 检查项 | 标准 | 状态 |
|------|--------|------|------|
| 1 | 语言标注 | 必须标注编程语言 | ☐ |
| 2 | 代码可运行 | 语法正确 | ☐ |
| 3 | 注释完整 | 关键步骤有注释 | ☐ |
| 4 | 长度适中 | 单块不超过50行 | ☐ |
| 5 | 缩进正确 | 4空格或Tab统一 | ☐ |

**代码块示例**：
```python
# 导入库
import requests

# 发送请求
response = requests.get('https://api.example.com')

# 返回结果
print(response.json())
```

### 2.4 引用检查

| 序号 | 检查项 | 标准 | 状态 |
|------|--------|------|------|
| 1 | 引用格式 | 使用 > 引用块 | ☐ |
| 2 | 来源标注 | 标注引用来源 | ☐ |
| 3 | 链接有效 | 外链可访问 | ☐ |
| 4 | 数量适中 | 不超过5个外链 | ☐ |
