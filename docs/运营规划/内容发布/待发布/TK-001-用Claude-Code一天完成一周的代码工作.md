## 三、怎么用？（实操）

### 1. 环境准备

**安装 Claude Code：**

```bash
# 使用 npm 安装
npm install -g @anthropic-ai/claude-code

# 或使用官方安装脚本
curl -fsSL https://claude.ai/install.sh | sh
```

**启动 Claude Code：**

```bash
# 在你的项目目录下启动
cd your-project
claude
```

**配置 API Key：**

首次使用需要配置 Anthropic API Key：

```bash
# 设置环境变量
export ANTHROPIC_API_KEY="your-api-key"

# 或在 ~/.claude/config.json 中配置
```

### 2. 基础用法

**场景1：让 Claude Code 帮你写代码**

```
你：帮我写一个用户登录的 API 接口，使用 Express.js

Claude：好的，我来帮你创建登录接口...

[创建文件 routes/auth.js]
[写入代码...]
[安装依赖 npm install jsonwebtoken bcryptjs]
```

### 3. 实战案例：一天完成一周的工作量

**背景**：产品经理周五下午说，下周一要上线一个"用户反馈系统"。

- 周一上午：联调测试

**使用 Claude Code**：

```markdown

你：我要做一个用户反馈系统，包含以下功能：
1. 用户提交反馈（文字+截图）
2. 管理员查看和回复反馈
3. 反馈状态管理（待处理/处理中/已解决）
技术栈：Next.js + Prisma + PostgreSQL

Claude：好的，我来帮你规划...

[创建数据库模型 schema.prisma]
[创建 API 路由]
[生成基础 CRUD 操作]

## 上午 10:30 - 核心功能

你：继续，添加图片上传功能，使用 S3 存储

Claude：好的，我来添加图片上传功能...

[配置 AWS S3]
[创建上传组件]
[添加图片预览功能]

## 下午 2:00 - 前端页面

你：帮我写反馈列表页面，要支持筛选和搜索

Claude：好的，我来创建反馈管理页面...

[创建页面组件]
```

### 5. 进阶技巧

**技巧1：使用 Skills 扩展能力**

Claude Code 支持 Skills 机制，可以自定义专业能力：

```markdown
# .claude/skills/code-review/SKILL.md

## 能力
你是一名资深代码审查专家，专注于：
- 代码质量检查
- 安全漏洞扫描
- 性能问题发现
- 最佳实践建议
```

**技巧2：利用 Memory 记住项目上下文**

```
你：记住，这个项目使用 TypeScript，数据库是 PostgreSQL

Claude：好的，我已经记住了这个项目的技术栈配置。

[后续对话中会自动遵循这些约定]
```

**技巧3：并行任务处理**

```
你：同时帮我做这三件事：
1. 检查所有 API 的错误处理
2. 为每个接口添加参数校验
3. 更新 API 文档

Claude：好的，我来并行处理这些任务...
[同时分析和修改多个文件]
```
