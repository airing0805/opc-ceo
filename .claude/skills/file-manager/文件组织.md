# 文件组织

## 功能概述

定义文件分类规则、命名约定和目录结构，帮助组织文件系统。

## 目录结构规范

### 标准目录结构

```
OPC-CEO/
├── .claude/              # Claude Code 配置和技能
│   ├── skills/          # 技能目录
│   └── agents/          # Agent 配置
├── plans/               # 规划和设计文档
│   └── v1-技能规划/
├── src/                  # 源代码
├── docs/                # 文档目录（中文命名）
├── tmp/                 # 临时文件
└── backups/              # 备份文件
```

### 文档目录结构

```
E:\Documents\
├── 项目文档/
│   ├── OPC-CEO/
│   └── 其他项目/
├── 技术文档/
├── 设计文档/
├── 会议记录/
└── 个人笔记/
```

## 文件命名规范

### 中文命名（文档文件）

**规则**：使用中文命名，简洁明了

| 类型 | 命名格式 | 示例 |
|------|----------|------|
| 设计文档 | `设计意图.md` | ✅ 正确 |
| 规划文档 | `规划文档.md` | ✅ 正确 |
| 角色设计 | `角色设计.md` | ✅ 正确 |
| 会议记录 | `会议记录-2026-02-23.md` | ✅ 正确 |

**避免**：
- ❌ `design_intention.md` - 文档不应使用英文
- ❌ `RoleDesign.md` - 文档不应使用英文
- ❌ `会议记录_final_v2.md` - 避免版本号后缀

### 代码命名（遵循语言规范）

**Python**: snake_case
- ✅ `task_manager.py`
- ✅ `file_operations.py`
- ❌ `TaskManager.py`

**TypeScript/JavaScript**: camelCase
- ✅ `taskManager.ts`
- ✅ `fileOperations.ts`
- ❌ `task_manager.ts`

**Go**: camelCase (导出) / snake_case (内部)
- ✅ `TaskManager` (导出)
- ✅ `task_manager` (内部)

### 临时文件

**规则**：放在 `tmp/` 目录，包含日期

```
tmp/
├── 思考古迹-2026-02-23.md
├── 快速笔记-2026-02-23.md
└── 测试草稿-2026-02-23.md
```

## 文件分类

### 按用途分类

| 分类 | 存放位置 | 文件类型 |
|------|----------|----------|
| 项目文档 | `docs/项目文档/` | 需求、设计、测试文档 |
| 技术文档 | `docs/技术文档/` | API文档、架构文档 |
| 会议记录 | `docs/会议记录/` | 会议纪要、决策记录 |
| 个人笔记 | `docs/个人笔记/` | 学习笔记、想法记录 |

### 按状态分类

| 状态 | 标识方式 | 说明 |
|------|----------|------|
| 进行中 | 文件名包含 `进行中` | 正在处理的文档 |
| 已完成 | 文件名包含 `已` | 已完成的文档 |
| 待审核 | 文件名包含 `待审` | 等待审核的文档 |
| 归档 | 放在 `archive/` 目录 | 不再活跃的文档 |

## 组织命令

### 命令 1：分析目录结构

**用途**：分析当前目录结构，给出优化建议

**语法**：
```
分析目录结构:
  - directory: <目录路径> (可选，默认工作目录)
```

**返回**：
- 目录树结构
- 文件类型分布
- 重复文件检测
- 优化建议

**示例**：
```
分析目录结构:
  - directory: .
```

---

### 命令 2：整理目录

**用途**：按分类规则整理目录

**语法**：
```
整理目录:
  - directory: <目录路径>
  - rules: <整理规则>
```

**整理规则**：

| 规则 | 说明 |
|------|------|
| 按类型 | 按文件类型分类 |
| 按日期 | 按创建/修改日期分类 |
| 按项目 | 按项目归属分类 |
| 按大小 | 按文件大小分类 |

**示例**：
```
整理目录:
  - directory: E:\Documents
  - rules: 按类型
```

---

### 命令 3：归档文件

**用途**：将不再活跃的文件移动到归档目录

**语法**：
```
归档文件:
  - path: <文件路径>
  - archiveDir: <归档目录> (可选)
```

**示例**：
```
归档文件:
  - path: E:\Documents\旧项目文档.md
  - archiveDir: E:\Documents\archive
```
