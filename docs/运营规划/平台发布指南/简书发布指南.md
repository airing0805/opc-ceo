## 二、标题优化技巧

### 简书标题公式

```
公式1：教程式
[动词] + [技术/工具] + [效果/目的]
例：用Claude Code构建Python项目，效率提升200%

公式2：问题式
如何 + [做什么] + [补充说明]
例：如何在2026年用AI提升编程效率？Claude Code实战指南

公式3：清单式
[数字] + [形容词] + [内容类型]
例：10个Claude Code实用技巧，每个程序员都应该知道

公式4：入门式
[技术/工具] + 入门 + [补充说明]
例：Claude Code入门：从零开始学AI编程

公式5：进阶式
[技术/工具] + 进阶 + [补充说明]
例：Claude Code进阶：打造你的AI编程工作流
```

### 标题禁忌
- ❌ 夸大标题：与内容不符的夸张表述
- ❌ 纯数字："第1篇"、"第2篇"无意义序号
- ❌ 过于简单：单字或词语作为标题
- ❌ 敏感内容：政治、争议、灰色地带

### 标题长度建议
| 类型 | 推荐长度 | 说明 |
|------|----------|------|
| 技术教程 | 15-25字 | 清晰说明内容 |
| 工具推荐 | 15-20字 | 突出工具价值 |
| 经验分享 | 12-20字 | 引发共鸣 |
| 深度分析 | 20-30字 | 体现专业性 |

### 代码块规范

#### 必须标注语言
```markdown
# 正确 ✅
```python
def hello():
    print("Hello, Claude Code!")
```

# 错误 ❌
```
def hello():
    print("Hello, Claude Code!")
```
```

#### 代码块最佳实践
| 规范 | 说明 |
|------|------|
| 语言标注 | 必须标注，支持语法高亮 |
| 代码注释 | 关键代码添加注释 |
| 行数控制 | 单个代码块不超过50行 |
| 完整性 | 代码可运行，包含必要依赖 |
| 缩进 | 统一使用4空格或Tab |

### 排版规范

| 元素 | 规范 | 示例 |
|------|------|------|
| 标题层级 | 使用#标记，最多4级 | # ## ### #### |
| 段落 | 空行分隔，每段3-5行 | - |
| 强调 | **加粗** 或 *斜体* | 重点内容 |
| 列表 | 使用 - 或 1. | - 无序 / 1. 有序 |
| 引用 | 使用 > 标记 | > 引用内容 |
| 分割线 | 使用 --- | --- |
| 表格 | 标准Markdown格式 | 见下方 |

```markdown
| 列1 | 列2 | 列3 |
|------|------|------|
| 内容 | 内容 | 内容 |
```

## 四、封面与配图要求

### 封面图规范
| 项目 | 要求 |
|------|------|
| 尺寸 | 1280x720px（16:9）或 800x600px（4:3） |
| 格式 | JPG/PNG/WebP |
| 大小 | <2MB |
| 位置 | 文章第一张图自动作为封面 |

### 配图类型与使用场景

| 类型 | 场景 | 说明 |
|------|------|------|
| 代码截图 | 展示运行结果 | 高亮关键输出 |
| 流程图 | 解释复杂流程 | 使用Mermaid或制图工具 |
| 架构图 | 系统设计说明 | 清晰展示组件关系 |
| 操作截图 | 教程步骤 | 标注操作位置 |
| 对比图 | 方案对比 | 直观展示差异 |

### 技术文章配图建议
1. **代码运行结果**：必须配截图证明
2. **复杂流程**：用流程图解释
3. **界面操作**：标注操作步骤
4. **数据对比**：用表格或图表

### 图片处理规范
- 使用 PNG 格式保留清晰度
- 代码截图使用等宽字体
- 重要区域用红色框标注
- 图片居中，添加说明文字

### 标签选择原则
1. **精准匹配**：标签与内容高度相关

## 九、常见问题

### Q1：文章如何上首页推荐？
**条件**：
1. 内容原创且高质量
2. 排版规范，可读性强
3. 被热门专题收录
4. 互动数据好（阅读、点赞、评论）

### Q2：如何提高文章收录率？
**方法**：
1. 选择匹配度高的专题
2. 遵守专题收录规则
3. 内容质量过硬
4. 持续投稿，建立信任

### Q3：付费专栏如何开通？
**条件**：
1. 账号等级达到要求
2. 有一定数量的优质文章
3. 申请并通过审核
