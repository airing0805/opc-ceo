## 一、追踪表说明

### 1.1 用途
- 追踪所有内容的发布状态
- 记录各平台发布链接和数据
- 支持数据采集流程
- 为内容策略优化提供数据基础

### 1.2 更新规则
- **发布后**：24小时内更新发布链接和时间
- **数据采集**：每周日晚汇总阅读量、点赞数、评论数
- **状态变更**：实时更新
- **新增内容**：内容完成后立即添加到追踪表

### 1.3 状态说明

| 标记 | 状态 | 说明 |
|------|------|------|
| `pending` | 待发布 | 内容已完成，等待发布 |
| `published` | 已发布 | 内容已发布，链接可用 |
| `tracking` | 已追踪 | 已采集首次数据 |
| `failed` | 发布失败 | 发布过程中遇到问题 |
| `removed` | 已下架 | 内容被平台下架 |

### 1.4 追踪字段定义

| 字段 | 说明 | 更新时机 |
|------|------|----------|
| 文章ID | 唯一标识符（如 TK-001, ZC-001） | 创建时 |
| 标题 | 文章完整标题 | 创建时 |
| 发布平台 | 知乎/CSDN/简书/掘金/小红书 | 发布时 |
| 发布日期 | 实际发布日期 | 发布时 |
| 发布链接 | 平台文章链接 | 发布后24h内 |
| 阅读量 | 首次采集数据 | 发布后7天内 |
| 点赞数 | 首次采集数据 | 发布后7天内 |
| 评论数 | 首次采集数据 | 发布后7天内 |
| 收藏数 | 首次采集数据 | 发布后7天内 |
| 状态 | pending/published/tracking | 实时更新 |

## 五、发布后记录流程

### 5.1 发布后24小时内

```
1. 发布完成后，立即记录：
   - 发布平台
   - 发布日期
   - 发布链接（复制完整URL）
   
2. 更新发布追踪表：
   - 找到对应文章ID行
   - 更新状态为 "published"
   - 填入发布日期和链接
   
3. 通知CEO确认记录完成
```

### 5.2 数据采集对接

```
发布后第7天：
1. 登录各平台后台
2. 采集首次数据：阅读量、点赞、评论、收藏
3. 更新追踪表对应字段
4. 状态更新为 "tracking"

后续采集：
- 每周日晚采集一次
- 记录增长变化
- 分析数据趋势
```
