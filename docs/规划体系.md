# OPC-CEO 规划体系

## 核心原则

**一切运营沟通必须通过 MD 文档，留下痕迹供用户查看。**

---

## 规划体系架构

### 四层规划架构

```
战略规划（长期，6-12月）
    │
    ├─→ 版本规划（中期，2-4周）
    │      │
    │      ├─→ 运营规划（短期，周）
    │      │      │
    │      │      └─→ 每日执行（工作日志）
    │      │
    │      └─→ 技术规划（专项）
    │
    └─→ 能力矩阵（角色成长）
```

---

## 规划文档分类

| 类型 | 时间跨度 | 用途 | 示例 |
|------|----------|------|------|
| 战略规划 | 6-12月 | 长期愿景和方向 | 公司愿景、战略目标 |
| 版本规划 | 2-4周 | 功能迭代和能力提升 | v2-技能规划、v3-能力提升 |
| 运营规划 | 周/月 | 日常运营和任务安排 | 本周计划、本月计划 |
| 技术规划 | 专项 | 技术架构和债务管理 | 架构演进、技术债务 |

---

## 任务优先级

| 级别 | 紧急度 | 重要性 | 响应时间 |
|------|--------|--------|----------|
| P0 | 高 | 高 | 立即 |
| P1 | 高/中 | 高 | 当天 |
| P2 | 中 | 中 | 本周 |
| P3 | 低 | 低 | 有空时 |

---

## 规划文档生命周期

1. **创建**：使用对应模板创建
2. **更新**：每次完成任务/沟通后更新
3. **归档**：规划周期结束后移至归档目录
4. **废弃**：记录废弃原因和替代方案
