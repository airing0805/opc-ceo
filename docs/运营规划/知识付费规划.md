### 2.2 模板 1：项目配置模板（CLAUDE.md）

#### 2.2.1 模板名称
Claude Code 项目配置文件

#### 2.2.2 核心功能
为 Claude Code 提供项目级上下文和配置指导，使其理解项目目的、设计原则和默认配置。

#### 2.2.3 包含的字段/模块

| 字段/模块 | 类型 | 必要性 | 说明 |
|-----------|------|--------|------|
| 项目概述 | 文本 | 必须 | 一句话描述项目目标和定位 |
| 设计原则 | 列表 | 必须 | 3-5 条核心设计原则 |
| 核心架构 | 架构图/文本 | 推荐 | 系统架构设计 |
| 实体 ID 命名规范 | 表格 | 推荐 | 任务/事件/对话等 ID 格式 |
| 默认配置 | 表格 | 必须 | Git 目录、命名规范等配置 |
| Skill 创建规范 | 文本 | 推荐 | Skill 文件命名和结构规范 |
| 实现状态 | 表格 | 推荐 | 各模块当前实现状态 |
| 关键文档 | 表格 | 推荐 | 重要文档索引 |
| currentDate | 日期 | 必须 | 当前日期上下文 |

#### 2.2.4 使用场景
- 新项目初始化时配置 Claude Code
- 切换项目时提供上下文
- 引导 Claude 理解项目规范

#### 2.2.5 示例结构
```markdown
# CLAUDE.md

## 项目概述
**{项目名称}** 是 {一句话描述}，基于 Claude Code Skills/Agents 实现。

## 设计原则
1. **原则一** - 描述
2. **原则二** - 描述
3. **原则三** - 描述

## 默认配置
| 配置项 | 值 |
|--------|-----|
| Git 项目目录 | `{你的项目目录}` |
| 文档命名规范 | 中文命名 |
| 临时目录 | `tmp/` |

## currentDate
Today's date is {YYYY-MM-DD}.
```

### 2.3 模板 2：技能定义模板（SKILL.md）

#### 2.3.1 模板名称
Claude Code Skill 定义模板

#### 2.3.2 核心功能
定义一个完整的 Skill，包含元数据、能力边界、模块索引和使用示例。

#### 2.3.3 包含的字段/模块

| 字段/模块 | 类型 | 必要性 | 说明 |
|-----------|------|--------|------|
| YAML Front Matter | 元数据 | 必须 | name, description, version, author, modules |
| 角色定位 | 文本 | 必须 | 角色职责和边界 |
| 能力要求 | 列表 | 必须 | 核心能力清单 |
| 默认配置 | 表格 | 推荐 | 可配置参数 |
| 模块索引 | 表格 | 必须 | 各模块说明 |
| 数据模型 | 实体定义 | 推荐 | 实体和关系定义 |
| 使用示例 | 示例代码 | 必须 | 典型使用场景 |
| 最佳实践 | 列表 | 推荐 | 使用建议 |
| 变更历史 | 表格 | 推荐 | 版本变更记录 |

#### 2.3.4 使用场景
- 创建新的 Skill
- 规范化 Skill 定义
- 复用已有 Skill 结构

#### 2.3.5 示例结构
```markdown
---
name: {skill-name}
description: {技能描述}
version: 1.0.0
source: local-creation
author: Your-Name
modules:
  - {模块1}
  - {模块2}
---

# {技能名称}

## 角色定位
[角色描述]

## 能力要求
- 能力1
- 能力2

## 模块索引
| 模块 | 说明 |
|------|------|
| 模块1 | 说明 |
| 模块2 | 说明 |

## 使用示例
### 示例场景
[具体示例]
```

### 2.4 模板 3：Agent 协作模板（agent-config.md）

#### 2.4.1 模板名称
多 Agent 协作配置模板

#### 2.4.2 核心功能

#### 2.4.3 包含的字段/模块

| 字段/模块 | 类型 | 必要性 | 说明 |
|-----------|------|--------|------|
| 基本信息 | 表格 | 必须 | Agent ID、名称、类型、调用方式 |
| 角色定义 | 文本 | 必须 | 核心职责描述 |
| 能力边界 | 表格 | 必须 | 能做/不能做 |
| 协作关系 | 表格 | 推荐 | 与其他 Agent 的协作方式 |
| 配置参数 | 表格 | 推荐 | 可调参数和默认值 |
| 调用示例 | 代码 | 必须 | 基础调用和带参数调用示例 |
| 注意事项 | 列表 | 推荐 | 使用注意事项 |

#### 2.4.4 使用场景
- 定义新的 Agent 角色
- 配置 Agent 间的协作
- 文档化 Agent 调用规范

#### 2.4.5 示例结构
```markdown
# Agent 配置 - {Agent 名称}

## 基本信息
| 项目 | 内容 |
|------|------|
| Agent ID | `{agent-id}` |
| 名称 | {中文名称} |
| 类型 | {skill/assistant/custom} |
| 调用方式 | {SendMessage/Team/直接调用} |

## 角色定义
### 核心职责
[职责描述]

### 能力边界
| 能做 | 不能做 |
|------|--------|
| xxx | yyy |

## 调用示例
```javascript
await sendMessage({
  to: "{agent-id}",
  message: "{消息内容}"
});
```
```

### 2.7 模板 6：任务管理模板（任务操作.md）

#### 2.7.1 模板名称
任务生命周期管理模板

#### 2.7.2 核心功能
定义任务从创建到完成的全流程管理规范。

#### 2.7.3 包含的字段/模块

| 字段/模块 | 类型 | 必要性 | 说明 |
|-----------|------|--------|------|
| 任务生命周期 | 流程图 | 必须 | 创建→分配→执行→验收→完成 |
| 任务实体模型 | 字段定义 | 必须 | 必要字段和可选字段 |
| 任务状态 | 枚举 | 必须 | 各状态定义和流转规则 |
| 操作流程 | 步骤说明 | 必须 | 创建/更新/验收等操作步骤 |
| ID 命名规范 | 格式 | 必须 | 任务 ID 格式 |

#### 2.7.4 使用场景
- 任务系统设计
- 任务管理流程规范化
- 任务追踪和统计

#### 2.7.5 示例结构
```markdown
# 任务操作

## 任务生命周期
创建 → 分配 → 执行 → 验收 → 完成/归档

## 任务实体模型
### 必要字段
| 字段 | 类型 | 说明 |
|------|------|------|
| taskId | string | 唯一标识 |
| title | string | 任务标题 |
| status | enum | 状态 |
| priority | enum | 优先级 |

## 任务操作流程
### 1. 创建任务
1. 验证任务信息完整性
2. 生成任务 ID
3. 创建任务实体

### 2. 更新任务状态
状态流转: 待执行 → 进行中 → 待验收 → 已完成
```
