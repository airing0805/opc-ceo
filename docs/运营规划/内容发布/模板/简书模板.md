## 二、标题公式

### 2.1 热门标题特点

简书技术类热门标题通常包含以下元素：

1. **明确受众**：告诉读者"这是写给谁看的"
2. **突出价值**：读者能获得什么
3. **制造好奇**：引起点击欲望
4. **强调时效**：暗示内容的时效性

### 2.2 标题公式模板

```
公式1：{目标受众}必看：{主题}完整指南
公式2：{数字}分钟学会{技能}，{附加价值}
公式3：{痛点问题}？一文讲透{解决方案}
公式4：{年份}最新{主题}：从入门到实战
公式5：{对比概念} vs {对比概念}：{结论}
公式6：{工具/技术}实战：{具体场景}全流程
```

### 2.3 标题示例

| 公式 | 示例标题 |
|------|----------|
| 公式1 | 前端开发者必看：TypeScript 完整指南 |
| 公式2 | 10分钟学会Docker部署，效率提升10倍 |
| 公式3 | API接口总是报错？一文讲透RESTful设计规范 |
| 公式4 | 2026年最新Git工作流：从入门到实战 |
| 公式5 | React vs Vue：大厂项目如何选择 |
| 公式6 | Claude Code实战：MCP开发全流程 |

### 2.4 标题禁忌

- ❌ 标题党（与内容不符）
- ❌ 过于抽象（"关于编程的一些想法"）
- ❌ 太长（超过30字）
- ❌ 全英文（简书用户中文为主）
- ❌ 夸大效果（"学会这个年薪百万"）

### 4.3 写作技巧

#### 段落规范

- 每段不超过 4-5 行
- 适当使用空行分隔
- 长内容用列表或表格

#### 代码规范

- 添加语言标识（\`\`\`javascript）
- 关键代码加注释
- 完整可运行的代码块

#### 强调重点

- **加粗**：关键概念、重要结论
- `代码格式`：命令、变量名、文件名
- > 引用：提示、警告、补充说明

## 一、MCP是什么？

**MCP = Model Context Protocol（模型上下文协议）**

它是 Claude Code 用来「连接外部世界」的桥梁。

### 打个比方

想象一下：

- **没有MCP**：你有一个超级聪明的秘书（Claude），但她只能看办公室里的文件，外面的世界一概不知
- **有了MCP**：这个秘书有了手机、电脑、汽车，可以帮你查资料、发邮件、管文件

MCP 就是给 AI 配的「工具套装」。

---

## 二、为什么要学MCP？

### 以前的问题

AI 再厉害，也有短板：

- ❌ 不会查实时天气
- ❌ 不会操作你的文件
- ❌ 不会调用你的数据库
- ❌ 不会发邮件、搜网页

### MCP 带来的改变

现在，AI 可以直接：

- ✅ 查天气、搜资料、读写文件
- ✅ 操作数据库、发送邮件

**一句话：AI 从「只会聊」变成了「真能干」。**

---

## 三、如何安装MCP？

### 步骤1：安装第一个MCP

以 GitHub MCP 为例：

```bash
# 安装 claude-github MCP
npx @anthropic-ai/claude-github-mcp
```

安装完成后，Claude Code 就能：

- 帮你读 GitHub Issues
- 创建/关闭 Issue
- 搜代码库
- 查看 PR 状态

### 步骤2：配置MCP

在 Claude Code 配置文件中添加：

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["@anthropic-ai/claude-github-mcp"]
    }
  }
}
```

### 步骤3：验证安装

重启 Claude Code，输入：

> 「帮我查看 GitHub 上最近的 Issue」

如果正常返回，说明安装成功！
