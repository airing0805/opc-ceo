## 二、采集指标定义

### 核心指标（每篇内容必采）

| 指标名称 | 说明 | 单位 |
|---------|------|------|
| **阅读量** | 内容被浏览的次数 | 次 |
| **点赞数** | 用户点赞数量 | 个 |
| **评论数** | 用户评论数量 | 条 |
| **收藏数** | 用户收藏数量 | 个 |
| **转发/分享数** | 用户转发或分享次数 | 次 |

### 账号指标（每日必采）

| 指标名称 | 说明 | 单位 |
|---------|------|------|
| **总粉丝数** | 账号累计粉丝 | 人 |
| **新增粉丝** | 当日新增粉丝 | 人 |
| **总关注数** | 账号关注他人数量 | 人 |

### 衍生指标（自动计算）

| 指标名称 | 计算公式 | 说明 |
|---------|---------|------|
| **互动率** | (点赞+评论+收藏+转发) / 阅读量 × 100% | 内容质量指标 |
| **涨粉率** | 新增粉丝 / 总粉丝 × 100% | 账号增长指标 |

## 四、采集步骤

### 标准采集流程

#### 第一步：准备工作

1. **打开采集工具**
   - 准备好数据采集模板（见 `数据采集模板.md`）
   - 或打开在线表格/Excel

2. **确认浏览器状态**
   - 确保已登录各平台账号
   - 如未登录，先完成登录（一次性）

3. **记录采集日期**
   - 在模板中填写当日日期
   - 格式：YYYY-MM-DD（如 2026-02-26）

#### 第二步：逐平台采集

**按顺序采集：知乎 → 简书 → CSDN → 掘金**

##### 知乎采集步骤

```
1. 打开知乎创作中心
   └─ 访问：https://www.zhihu.com/creator/content-management/all

2. 查看内容数据
   ├─ 找到已发布的文章/回答
   ├─ 记录每篇内容的：阅读量、点赞、评论、收藏
   └─ 如有新增内容，先添加到模板

3. 查看账号数据
   ├─ 进入「数据中心」
   ├─ 记录：总粉丝数、新增粉丝
   └─ 退出当前页面

4. 填写采集时间戳
   └─ 记录完成时间（如 21:35）
```

##### 简书采集步骤

```
1. 打开简书后台
   └─ 访问：https://www.jianshu.com/writer#/stats

2. 查看文章统计
   ├─ 找到已发布的文章
   ├─ 记录每篇内容的：阅读量、点赞、评论
   └─ 简书无收藏数据，跳过

3. 查看账号数据
   ├─ 点击头像进入个人主页
   ├─ 记录：总关注者数（粉丝）
   └─ 退出当前页面

4. 填写采集时间戳
   └─ 记录完成时间
```

##### CSDN 采集步骤

```
1. 打开 CSDN 创作者中心
   └─ 访问：https://mp.csdn.net/mp_blog/creation/success

2. 查看文章数据
   ├─ 找到已发布的博客
   ├─ 记录每篇内容的：阅读量、点赞、评论、收藏
   └─ 如有新增，添加到模板

3. 查看账号数据
   ├─ 进入「数据中心」
   ├─ 记录：总粉丝数、新增粉丝
   └─ 退出当前页面

4. 填写采集时间戳
   └─ 记录完成时间
```

##### 掘金采集步骤

```
1. 打开掘金创作者中心
   └─ 访问：https://juejin.cn/creator/content

2. 查看文章数据
   ├─ 找到已发布的文章
   ├─ 记录每篇内容的：阅读量、点赞、评论、收藏
   └─ 如有新增，添加到模板

3. 查看账号数据
   ├─ 进入「数据中心」或个人主页
   ├─ 记录：总粉丝数、新增粉丝
   └─ 退出当前页面

4. 填写采集时间戳
   └─ 记录完成时间
```

#### 第三步：数据验证

```
1. 检查完整性
   ├─ 确认所有必采字段已填写
   ├─ 确认没有空值或异常值
   └─ 确认日期时间正确

2. 简单校验
   ├─ 阅读量应 ≥ 0
   ├─ 新增粉丝数应 ≥ 0（或为负，如果掉粉）
   └─ 互动指标应 ≤ 阅读量

3. 保存数据
   ├─ 如果使用 Excel/表格，保存文件
   ├─ 如果使用模板，提交表单
   └─ 备份到云盘（可选）
```

## 五、采集工具/方法说明

### 推荐采集方式

#### 方式一：模板记录（推荐初期使用）

- **工具**: `数据采集模板.md`（已创建）
- **优点**: 标准化、不易遗漏
- **缺点**: 需要手动填写
- **适用**: 采集初期，数据量少

#### 方式二：Excel/在线表格

- **工具**: Excel / Google Sheets / 腾讯文档
- **优点**: 便于数据分析、自动计算
- **缺点**: 需要自行设计表格结构
- **适用**: 数据量增多后

#### 方式三：自动化采集（未来）

- **工具**: Python + Playwright / Puppeteer
- **优点**: 自动化、节省时间
- **缺点**: 开发成本高、需维护
- **适用**: 数据采集稳定后

| 浏览器 | 访问平台数据页 | ✅ 已准备 |
| 数据采集模板 | 记录数据 | ✅ 已创建 |
| Excel/表格 | 数据汇总分析 | ✅ 可用 |

## 六、数据录入指引

### 录入规范

#### 日期格式

```
统一格式：YYYY-MM-DD
示例：2026-02-26
```

#### 时间格式

```
统一格式：HH:MM（24 小时制）
示例：21:35、09:15
```

#### 数字格式

```
整数：直接填写（如 1234）
百分比：保留 2 位小数（如 12.34%）
不记录小数点后的无效数字
```

#### 空值处理

```
如某项数据无法获取：
├─ 记录为 "N/A"（Not Available）
├─ 不要留空
└─ 备注说明原因
```

### 录入检查清单

每日采集完成后，确认以下项：

- [ ] 所有平台的**每篇内容**都已记录
- [ ] 所有**核心指标**（阅读、点赞、评论、收藏）已填写
- [ ] 所有平台的**账号指标**（粉丝数）已记录
- [ ] 采集日期和时间已填写
- [ ] 没有异常值（如阅读量为负数）
- [ ] 数据已保存/备份

### 异常情况处理

| 异常情况 | 处理方式 |
|---------|---------|
| 平台无法访问 | 记录 "N/A"，备注原因，稍后重试 |
| 数据页改版 | 记录问题，调整采集路径 |
| 数据明显异常 | 截图保存，记录备注，继续采集 |
| 忘记采集 | 次日补采，标注 "补采" |

## 七、每周汇总

### 周数据汇总时间

```
每周一 10:00 - 11:00
├─ 汇总上周（周一至周日）数据
├─ 分析趋势和异常
└─ 生成周报（见 `周数据汇总机制.md`）
```

### 汇总内容

1. **各平台总览**
   - 累计阅读量
   - 累计互动量
   - 新增粉丝数

2. **内容表现**
   - 爆款内容（阅读量 TOP 3）
   - 高互动内容（互动率 TOP 3）
   - 新发布内容数据

3. **趋势分析**
   - 数据增长趋势
   - 平台对比分析
   - 优化建议

## 八、注意事项

### 采集纪律

1. **坚持每日采集**
   - 养成固定习惯
   - 设置提醒闹钟
   - 不要中断

2. **数据真实性**
   - 如实记录，不篡改
   - 异常数据要备注
   - 保持数据完整性

3. **及时性**
   - 当日数据当日采
   - 不要累积多日
   - 如遗忘，次日补采

### 数据安全

1. **备份习惯**
   - 每周备份一次数据文件
   - 使用云盘自动同步
   - 保留历史版本

2. **隐私保护**
   - 不要公开分享数据文件
   - 敏感信息要加密
   - 定期更换账号密码
