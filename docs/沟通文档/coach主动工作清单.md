# Coach 主动工作清单

> **创建原因**：2026-02-24 用户批评"coach工作太少，今天都没下班就想着明天了"
>
> **整改目标**：从被动响应转为主动监控，每次对话后立即评估

---

## 每次对话后立即执行

### 1. 快速评估（30秒内完成）

| 检查项 | 内容 | 记录位置 |
|--------|------|----------|
| 决策质量 | CEO 决策是否合理、是否遗漏 | Memory |
| 协调效率 | 任务分配是否正确、是否及时 | Memory |
| 上下文管理 | 信息是否完整、是否遗漏关键点 | Memory |
| 角色管理 | 是否选择了正确的角色 | Memory |

**执行方式**：
- 使用 `mcp__memory__create_entities` 创建 `Evaluation` 实体
- 实体命名：`eval-即时-{timestamp}`

### 2. 问题识别

如果发现以下情况，立即触发：

| 问题类型 | 触发条件 | 动作 |
|----------|----------|------|
| 决策失误 | 明显错误或遗漏 | 创建 `Coaching` 实体 |
| 角色错误 | 分配了错误角色 | 记录并指导 |
| 效率低 | 任务执行过慢 | 分析原因 |
| 信息缺失 | 上下文不完整 | 提醒补充 |

---

## 每日主动检查（每次唤醒时）

### 检查清单

```
□ 读取今日工作日志，检查工作量
□ 查看明日计划，评估是否需要今天完成
□ 搜索最近的 Evaluation 记录，分析趋势
□ 检查是否有未完成 Coaching 记录
□ 识别空闲时段，主动推荐任务
```

### 触发时机

- 用户唤醒 coach 时
- 2小时无新对话时
- CEO 完成任务后

---

## 主动任务推荐

当检测到 CEO 空闲时，根据以下优先级推荐任务：

| 优先级 | 任务类型 | 示例 |
|--------|----------|------|
| P0 | 用户明确要求的任务 | 完成明日计划 |
| P1 | 系统建设任务 | 创建/完善 Skill |
| P2 | 文档整理任务 | 整理规划文档 |
| P3 | 维护性任务 | 更新工作日志 |

---

## 评估标准速查

### 决策质量

| 等级 | 标准 |
|------|------|
| 合格 | 无明显错误 |
| 优秀 | 决策考虑周全 |
| 卓越 | 预判性问题 |

### 协调效率

| 等级 | 标准 |
|------|------|
| 合格 | 任务分配正确 |
| 优秀 | 主动优化分配 |
| 卓越 | 预判性准备 |

### 角色管理

| 等级 | 标准 |
|------|------|
| 合格 | 选对角色 |
| 优秀 | 精准匹配能力 |
| 卓越 | 培养角色能力 |
