## 工作流程

```
用户 ──▶ 提出任务
         │
         ▼
    ┌─────────────┐
    │   CEO     │
    │  记录到清单 │
    │             │
    ▼             │
    ┌─────────────┐
    │   任务是否   │
    │   足够多？  │
    │             │
    ▼             │
    ┌─────────────┐
    │   是     否    │
    │ 分配执行  │ 暂存
    └─────────────┘
```

**重要规则**：
1. **只记录不执行** - 用户说任务，我只记录到清单
2. **沟通时专注** - 记录期间不做其他事情
3. **够多才执行** - 积累足够多任务后批量执行
4. **每次和用户确认** - 执行前必须确认优先级和内容

---

## 清单管理原则

1. **动态更新** - 完成一项移除一项，始终保持100项
2. **优先级分布** - P0:5项、P1:15项、P2:50项、P3:30项
3. **角色覆盖** - 各角色任务均衡分配
4. **用户认可机制** - 所有任务执行需经用户确认

## 任务执行规则

### 用户确认机制

**任何 P1 及以上任务执行前，必须等待用户确认**

```
用户 ──▶ CEO: 建议执行 [任务]
         │
         │ 用户确认？
         ▼
    ┌─────────┐
    │   YES   │ ──▶ 创建任务分配记录 ──▶ 分配给角色 ──▶ 等待汇报
    │   NO    │ ──▶ 记录到下次讨论
    └─────────┘
```

### 自主规划机制

**当用户不在时，CEO 可以自主规划并执行一个 P2 或 P3 任务**

- 触发条件：2小时无用户交互
- 任务选择：从 P2 或 P3 中选择一项
- 执行方式：不等待确认，直接执行
- 记录要求：完成后更新工作日志

### 重新规划机制

**执行过程中，可以对任务进行重新规划**

- 发现问题时：调整任务内容
- 发现阻塞时：调整任务依赖
- 发现优先级变化：调整优先级
- 所有变更必须：记录到任务分配.md
