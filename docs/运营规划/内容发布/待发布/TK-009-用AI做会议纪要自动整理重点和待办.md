## 一、为什么需要AI做会议纪要？

### 1.1 传统会议纪要的痛点

| 痛点 | 具体表现 |
|------|---------|
| 记不住 | 开会时拼命记，会后看不懂 |
| 写太慢 | 边开会边记录，跟不上节奏 |
| 写不完 | 会后整理太费时间，拖着拖着就忘了 |
| 不完整 | 遗漏重要信息，事后找不到 |
| 没人看 | 写了没人看，下次开会同样问题再问 |
| 无跟进 | 待办事项散落各处，无人追踪 |

### 1.2 AI会议纪要的优势

| 维度 | 人工记录 | AI辅助 |
|------|---------|--------|
| 准确性 | 遗漏30%信息 | 几乎100%捕获 |
| 完整性 | 片段式记录 | 全文转录+摘要 |
| 时效性 | 会后1-2小时产出 | 会后5-10分钟产出 |
| 结构化 | 笔记杂乱 | 自动分类整理 |
| 可检索 | 纸质/散乱文档 | 全文可搜索 |
| 待办追踪 | 人工整理 | 自动提取+提醒 |

### 1.3 AI会议纪要的核心价值

1. **释放注意力**：开会专注讨论，不必担心漏记
2. **提升效率**：会后10分钟出纪要，立即分发
3. **确保完整**：所有信息都被记录，不会遗漏
4. **便于回溯**：随时搜索历史会议内容
5. **强化执行**：待办自动提取，跟进更到位

## 二、AI会议纪要完整流程

### 2.1 流程概览

```
会前准备 → 会中录音 → 会后转录 → AI整理 → 人工审核 → 分发存档
    ↓         ↓          ↓          ↓         ↓          ↓
  准备议程   AI实时转录   语音转文字  生成纪要   微调确认   同步跟进
```

### 2.2 会前准备

**准备清单**：

| 准备项 | 说明 |
|--------|------|
| 确定议程 | 明确会议要讨论的事项 |
| 选择工具 | 确定使用的录音/转录工具 |
| 测试设备 | 确保麦克风、网络正常 |
| 告知参会 | 如需录音，提前告知 |

**议程模板**：

```markdown
## 会议议程

**会议主题**：[主题]
**时间**：[日期时间]
**参会人员**：[名单]
**主持人**：[姓名]

### 讨论事项
1. [事项1] - 负责人：[姓名] - 预计10分钟
2. [事项2] - 负责人：[姓名] - 预计15分钟
3. [事项3] - 负责人：[姓名] - 预计10分钟

### 决策事项
- [需要决策的问题]

### 期望产出
- [会议要达成的目标]
```

### 2.3 会中录音

**三种方式**：

| 方式 | 工具 | 优点 | 缺点 |
|------|------|------|------|
| 实时转录 | Otter.ai/飞书妙记 | 即时可见，可互动 | 需要网络 |
| 录音+后期转录 | 讯飞听见/通义听悟 | 不受网络限制 | 会后才能看 |
| 线上会议自动 | 腾讯会议/Zoom | 无需额外工具 | 仅限线上会议 |

**录音注意事项**：

1. 提前告知参会人员会进行录音
2. 确保发言人声音清晰
3. 标注会议关键节点（重要决策时备注）
4. 录音文件及时命名和保存

### 2.4 AI整理

**核心Prompt模板**：

```
请将以下会议转录内容整理为结构化的会议纪要：

[转录内容]

整理要求：
1. **会议基本信息**：提取时间、参会人、主题
2. **讨论要点**：按议题分类，提炼核心观点
3. **决策结论**：明确记录达成的决策
4. **待办事项**：提取所有待办，格式为"任务 | 负责人 | 截止时间"
5. **遗留问题**：记录未解决、需要后续讨论的问题
6. **下一步行动**：明确后续安排

格式要求：
- 使用清晰的标题和小标题
- 每个议题用bullet points呈现
- 待办事项用表格形式
- 控制在1000字以内（去除口语化表达）
```

### 2.5 人工审核

**审核要点**：

| 检查项 | 说明 |
|--------|------|
| 准确性 | 关键数据、人名、时间是否正确 |
| 完整性 | 是否有遗漏的重要信息 |
| 待办明确 | 每个待办是否有人、有时限 |
| 敏感信息 | 是否需要删除敏感内容 |
| 格式规范 | 是否易于阅读和理解 |

### 2.6 分发存档

**分发流程**：

1. 通过邮件/群聊发送给参会者
2. 同步到团队知识库（Notion/飞书文档）
3. 待办事项同步到任务管理工具
4. 重要会议标记，方便后续检索

## 五、会议待办跟进最佳实践

### 5.1 待办追踪系统

**问题**：会议待办散落各处，没人跟进

**解决方案**：建立统一的待办追踪系统

```
会议纪要 → 提取待办 → 同步到任务工具 → 设置提醒 → 定期复盘
```

**推荐工具组合**：

| 工具 | 用途 |
|------|------|
| 飞书/钉钉 | 会议+待办一体化 |
| Notion | 会议纪要+任务看板 |
| Todoist/TickTick | 个人待办管理 |
| Make/Zapier | 自动同步待办 |

### 5.2 待办跟进模板

```markdown
## 会议待办追踪表

**会议名称**：[名称]
**会议日期**：[日期]

| 序号 | 待办事项 | 负责人 | 截止时间 | 状态 | 更新日期 | 备注 |
|------|---------|--------|----------|------|----------|------|
| 1 | [事项] | [姓名] | [日期] | 进行中 | [日期] | [说明] |
| 2 | [事项] | [姓名] | [日期] | 已完成 | [日期] | [说明] |
| 3 | [事项] | [姓名] | [日期] | 待开始 | - | [说明] |

### 状态说明
- 待开始：尚未开始
- 进行中：正在推进
- 已完成：任务完成
- 已延期：超过截止时间未完成
- 已取消：任务取消
```

### 5.3 定期复盘机制

**周度复盘**：

- 每周五回顾本周会议待办完成情况
- 延期事项分析原因，制定补救措施
- 更新待办状态

**月度复盘**：

- 统计会议待办完成率
- 分析高频延期原因
- 优化会议和待办管理流程

## 六、实战案例分享

### 案例1：产品团队周会

**背景**：20人产品团队，每周例会1.5小时

**原流程**：

- 指定一人记录，会后整理2小时
- 纪要不完整，待办跟进率低
- 经常"上次说了，这次怎么又问"

**新流程（AI辅助）**：

```
1. 会前：飞书日历自动创建会议+妙记
2. 会中：飞书妙记自动录音+实时转录
3. 会后：
   - AI自动生成纪要（5分钟）
   - 负责人审核微调（10分钟）
   - 待办自动同步到飞书任务
   - 纪要发布到团队群
```

**效果**：

- 纪要产出时间：2小时 → 15分钟
- 待办跟进率：40% → 90%
- 会议信息可随时搜索回溯

### 案例2：跨部门协调会

**背景**：多部门项目协调会，决策事项多

**挑战**：

- 参会人员多，观点各异
- 决策事项需要明确记录
- 会后各方对决策理解不一致

**解决方案**：

```
1. 使用Otter.ai实时转录
2. 会中关键决策时手动标注

```

**效果**：

- 决策记录清晰，无歧义
- 有据可查，避免扯皮
- 跨部门协作效率提升

## 七、常见问题解答

### Q1：录音会不会侵犯隐私？

**答**：需要注意：

1. 会前告知参会者会进行录音
2. 敏感会议可选择不录音
3. 录音文件妥善保存，设置访问权限
4. 纪要发布前删除敏感信息

### Q2：AI转录准确率够吗？

**答**：主流工具准确率已达到90-95%，但：

1. 专业术语可能识别错误，需要人工校对
2. 多人会议时说话人识别可能不准
3. 重要会议建议人工复核关键信息

### Q3：不用AI工具能做吗？

**答**：可以，用通用AI工具（Claude/ChatGPT）：

1. 自己录音+用工具转录（如讯飞）
2. 将转录文本发给AI，用Prompt整理
3. 效果一样好，只是多一步操作

### Q4：会议纪要要写多详细？

**答**：取决于会议类型：

| 会议类型 | 纪要详细程度 |
|----------|-------------|
| 日常例会 | 精简版，重点+待办 |
| 重要决策会 | 详细版，完整记录讨论过程 |
| 头脑风暴 | 全文保留，方便回顾创意 |
| 一对一沟通 | 要点记录即可 |
