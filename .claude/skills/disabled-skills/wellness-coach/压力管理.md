# 压力管理

## 功能概述

识别压力源，评估压力水平，提供应对策略和缓解技巧，帮助用户有效管理压力。

## 压力等级评估

| 等级 | 状态 | 表现特征 | 建议行动 |
|------|------|----------|----------|
| 1-3 | 正常 | 心态平和，应对自如 | 保持当前节奏，定期记录 |
| 4-5 | 轻度 | 偶尔紧张，略感压力 | 增加休息，注意调节情绪 |
| 6-7 | 中度 | 频繁紧张，影响效率 | 主动减压，调整工作计划 |
| 8-9 | 重度 | 持续焦虑，难以集中 | 暂停工作，寻求支持 |
| 10 | 危机 | 极度焦虑，濒临崩溃 | 建议寻求专业心理咨询师帮助 |

## 压力源识别

### 常见压力源分类

| 类别 | 具体压力源 | 典型表现 |
|------|-------------|----------|
| 工作相关 | 截止日期、客户需求、技术难题 | 焦虑、紧张、加班 |
| 决策压力 | 重大决策、不确定性 | 担忧、犹豫、失眠 |
| 财务压力 | 收入不稳定、成本控制 | 焦虑、担忧未来 |
| 孤独感 | 缺乏社交、独自承担 | 抑郁、情绪低落 |
| 角色冲突 | 多重身份、时间冲突 | 疲惫、混乱感 |
| 健康问题 | 身体不适、睡眠不足 | 易怒、注意力下降 |

### 创业者特有压力

一人公司创业者面临的独特压力：

1. **决策焦虑** - 所有重要决策都由自己承担
2. **角色过载** - 同时承担多个职责
3. **收入不稳定** - 没有固定薪资保障
4. **缺乏反馈** - 缺乏同事的日常支持和反馈
5. **边界模糊** - 工作与生活界限不清

## 应对策略

### 短期应对（即时缓解）

```
压力等级 4-5：
- 深呼吸 2-5 分钟
- 离开当前环境，走动 5 分钟
- 饮用温水，做简单拉伸

压力等级 6-7：
- 进行 10 分钟身体扫描冥想
- 切换到轻松的任务或活动
- 联系朋友或家人聊天

压力等级 8-9：
- 停止当前工作，彻底休息
- 进行 20-30 分钟正念练习
- 寻求社交支持

压力等级 10：
- 立即停止一切工作
- 考虑寻求专业心理咨询
- 优先恢复健康状态
```

### 长期管理（根本解决）

| 策略 | 具体方法 | 预期效果 |
|--------|----------|----------|
| 任务管理 | 合理规划、设置优先级、学会说"不" | 减少工作压力源 |
| 时间管理 | 番茄工作法、定期休息、设定边界 | 提高效率，减少加班 |
| 社交连接 | 定期联系朋友、加入社群、参加活动 | 缓解孤独感 |
| 身体健康 | 规律运动、充足睡眠、健康饮食 | 增强抗压能力 |
| 正念练习 | 每日冥想、呼吸练习、身体扫描 | 提升心理韧性 |

## 记录流程

### 创建压力事件记录

```
1. 用户报告当前压力状态
2. 创建 StressEvent 实体：
   - timestamp: 当前时间
   - stressLevel: 压力等级 (1-10)
   - source: 压力源类型
   - description: 事件详细描述
   - copingMethod: 应对方式
   - outcome: 处理结果
3. 关联到当日的 WellnessRecord
```

### 跟踪应对效果

记录每次压力事件后的应对方式及效果：

| 应对方式 | 效果评分 | 频率 |
|----------|----------|------|
| 深呼吸 | 有效/部分有效/无效 | 高频 |
| 运动 | 有效/部分有效/无效 | 中频 |
| 冥想 | 有效/部分有效/无效 | 低频 |

## 与 Memory 的交互

### 查询压力记录

```
使用 mcp__memory__search_nodes:
- entityType: StressEvent
- 按时间范围过滤
- 按压力等级过滤
- 按压力源分类
```

### 创建压力记录

```
使用 mcp__memory__create_entities:
- 创建 StressEvent 实体
- 创建 belongsTo 关系到 WellnessRecord
```

## 使用示例

### 示例 1：记录压力事件

```
用户: "压力很大，现在大概8分，主要是一个项目的截止日期快到了，还有技术问题没解决"

wellness-coach 处理:
1. 确认压力等级为8（重度），需要立即行动
2. 创建 StressEvent:
   - timestamp: 当前时间
   - stressLevel: 8
   - source: deadline, technical_issue
   - description: 项目截止日期临近 + 技术问题未解决
3. 提供立即建议：
   - "建议暂停工作，彻底休息一下"
   - "可以进行20分钟正念练习"
   - "如需要，可以寻求外部帮助"
```

### 示例 2：分析压力模式

```
用户: "我最近压力的主要来源是什么？"

wellness-coach 处理:
1. 查询最近一周的 StressEvent
2. 按 source 字段统计频率
3. 识别高频压力源
4. 提供针对性建议
```

### 示例 3：推荐应对策略

```
用户: "压力7分，我该怎么办？"

wellness-coach 处理:
1. 识别当前压力等级为7（中度）
2. 根据压力源推荐策略：
   - 如果是工作相关：建议调整任务优先级
   - 如果是决策压力：建议寻求建议或延后决定
   - 如果是孤独感：建议联系朋友或参加社群活动
3. 提供即时缓解方法：10分钟冥想或休息
```

## 预警机制

### 触发预警的条件

1. **连续高压** - 连续3天压力等级 ≥ 7
2. **压力激增** - 一日内压力等级变化 ≥ 5
3. **无效应对** - 相同压力源反复出现且应对方式无效

### 预警行动

当触发预警时：

1. **主动提醒** - 明确告知用户当前状态
2. **建议行动** - 提供具体可执行的建议
3. **记录追踪** - 记录预警事件以便后续分析
4. **必要时建议专业帮助** - 压力等级10时明确建议

## 最佳实践

1. **诚实评估** - 不低估压力等级，及时寻求帮助
2. **提前识别** - 留意压力累积的早期信号
3. **记录模式** - 帮助用户识别自己的压力模式
4. **多种策略** - 准备多种应对方式，灵活使用
5. **定期回顾** - 定期分析压力记录，调整策略
