## 索引说明

### 日志编号规范

执行日志使用以下编号格式：`EXEC-YYYY-MM-DD-NNN`

| 部分 | 说明 | 示例 |
|------|------|------|
| EXEC | 固定前缀 | EXEC |
| YYYY | 年份 | 2026 |
| MM | 月份 | 02 |
| DD | 日期 | 26 |
| NNN | 当日序号 | 001, 002, 003... |

### 状态标记

| 状态 | 符号 | 说明 |
|------|------|------|
| 全部完成 | ✅ | 所有目标达成 |
| 部分完成 | ⚠️ | 部分目标达成，有遗留问题 |
| 未完成 | ❌ | 主要目标未达成 |
| 进行中 | 🔄 | 执行尚未结束 |
| 已暂停 | ⏸️ | 临时暂停，等待条件恢复 |

## 查询方法

### 1. 按日期查询

找到对应月份的归档目录，查看该月份的日志文件。

示例：查找 2026年2月26日 的执行日志
```
路径：docs/沟通文档/执行日志/2026-02/
文件：EXEC-2026-02-26-NNN.md
```

### 2. 按任务 ID 查询

在本索引页面搜索任务 ID（如 `v8.3.6`），找到对应的日志记录。

### 3. 按执行人查询

在本索引页面搜索执行人名称（如 `CEO`、`Coach`）。

### 4. 按状态查询

在本索引页面搜索状态标记：
- 搜索 `✅` 查找已完成任务
- 搜索 `⚠️` 查找部分完成任务
- 搜索 `❌` 查找未完成任务

## 使用指南

### 创建新日志

1. **确认月份目录存在**
   - 检查 `docs/沟通文档/执行日志/YYYY-MM/` 目录
   - 如不存在，创建该目录

2. **复制模板**
   - 复制 `执行日志模板.md` 内容
   - 重命名为 `EXEC-YYYY-MM-DD-NNN.md`
   - NNN 为当日序号，从 001 开始

3. **填写日志**
   - 按照模板格式填写各部分内容
   - 确保时间戳、任务 ID 等信息准确

4. **更新索引**
   - 在本文件的对应月份表格中添加新记录
   - 更新统计数据

### 更新已有日志

1. 打开对应的日志文件
2. 更新执行步骤、决策、问题解决等内容
3. 更新最终状态和结果
4. 同步更新本索引中的状态
