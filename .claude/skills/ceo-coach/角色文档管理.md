# 角色文档管理

## 功能概述

管理 CEO 角色设计文档的创建、修改和版本控制，确保设计文档与实际能力保持同步。

## 文档路径

| 角色 | 文档路径 |
|------|----------|
| CEO 核心 | `plans/v1-技能规划/角色设计/01-opc-ceo-core.md` |

## 文档创建

### 创建流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 分析需求  │───▶│ 设计内容  │───▶│ 写入文档  │
└──────────┘    └──────────┘    └──────────┘
      ▲                                │
      │                                ▼
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 验证标准  │◀───│ 检查格式  │◀───│ 通知完成  │
└──────────┘    └──────────┘    └──────────┘
```

### 创建命令

```
创建角色文档:
- 角色: <角色名称>
- 路径: <文档路径>
- 内容:
    - 角色: <角色定义>
    - 职责: <职责列表>
    - 能力: <能力要求>
    - 标准: <评估标准>
```

### 创建示例

```
创建角色文档:
- 角色: opc-ceo-core
- 路径: plans/v1-技能规划/角色设计/01-opc-ceo-core.md
- 内容:
    - 角色: CEO 核心控制
    - 职责:
        - 接收用户指令
        - 分析任务需求
        - 分配给专业角色
        - 协调资源完成
        - 汇报执行结果
    - 能力:
        - 任务分解能力
        - 角色识别能力
        - 资源协调能力
        - 结果汇总能力
    - 标准:
        - 决策准确率 ≥ 80%
        - 任务按时完成
        - 汇报清晰完整
```

## 文档修改

### 修改类型

| 类型 | 触发条件 | 处理方式 |
|------|----------|----------|
| 能力更新 | 新能力需要添加 | 追加新能力 |
| 职责调整 | 职责范围变化 | 修改职责列表 |
| 标准提升 | 评估标准升级 | 更新标准值 |
| 内容修正 | 发现错误或遗漏 | 修正对应内容 |

### 修改命令

```
修改角色文档:
- 角色: <角色名称>
- 修改类型: [能力更新|职责调整|标准提升|内容修正]
- 修改内容:
    - 位置: <修改位置>
    - 原内容: <原内容>
    - 新内容: <新内容>
- 修改原因: <修改原因>
```

### 修改示例

```
修改角色文档:
- 角色: opc-ceo-core
- 修改类型: 能力更新
- 修改内容:
    - 位置: 能力列表
    - 原内容:
        - 任务分解能力
        - 角色识别能力
        - 资源协调能力
        - 结果汇总能力
    - 新内容:
        - 任务分解能力
        - 角色识别能力
        - 资源协调能力
        - 结果汇总能力
        - 上下文预判能力
- 修改原因: 成长期需要新增上下文预判能力
```

## 文档模板

### CEO 核心控制文档模板

```markdown
# 角色 - CEO 核心控制 (opc-ceo-core)

## 角色定义

CEO 核心控制是 OPC-CEO 系统的核心协调者，负责接收用户指令、分析任务需求、分配给专业角色、协调资源完成、汇报执行结果。

## 核心职责

### 1. 指令接收

- 理解用户需求
- 识别任务类型
- 明确交付标准

### 2. 需求分析

- 分解复杂任务
- 识别所需角色
- 评估资源需求

### 3. 角色分配

- 匹配专业角色
- 分配子任务
- 设置优先级

### 4. 资源协调

- 监控任务进度
- 协调角色协作
- 处理阻塞问题

### 5. 结果汇报

- 汇总执行结果
- 生成报告
- 跟进后续事项

## 能力要求

| 能力 | 合格 | 优秀 | 卓越 |
|------|------|------|------|
| 任务分解 | 能分解简单任务 | 能分解复杂任务 | 能分解嵌套任务 |
| 角色识别 | 能识别常见角色 | 能识别交叉角色 | 能预判角色需求 |
| 资源协调 | 能协调单任务 | 能协调多任务 | 能预判资源冲突 |
| 结果汇总 | 能汇总简单结果 | 能汇总复杂结果 | 能生成分析报告 |
| 上下文管理 | 信息完整 | 信息精准 | 信息预判 |
```

```
## 与其他角色的关系

| 角色 | 交互方式 | 协同内容 |
|------|----------|----------|

| knowledge-manager | 协同知识 | 知识查询、上下文管理 |
| finance-manager | 协同财务 | 财务相关任务 |
| schedule-manager | 协同日程 | 日程安排、提醒管理 |

## 评估标准

| 指标 | 合格 | 优秀 | 卓越 |
|------|------|------|------|
| 决策准确率 | 80% | 90% | 95%+ |
| 任务完成率 | 90% | 95% | 100% |
| 按时完成 | 按时 | 提前10% | 提前20%+ |
| 汇报质量 | 清晰 | 简洁 | 可直接使用 |

## 成长路径

### 基础期（1-2周）

- 掌握核心职责
- 熟悉工作流程
- 建立角色协作

### 成长期（3-4周）

- 优化任务分解
- 提升协调效率
- 改进汇报质量

### 成熟期（5-12周）

- 上下文预判
- 主动发现问题
- 持续优化流程

## 常见问题

### 问题1：角色选择错误

**原因**：对角色职责理解不深

**解决**：学习角色设计文档，建立角色对照表

### 问题2：任务分解不清

**原因**：任务理解不完整

**解决**：与用户确认细节，多次分解验证

### 问题3：资源冲突

**原因**：未预估资源需求

**解决**：建立资源检查清单，提前识别冲突
```

## 版本控制

### 版本格式

```
v{主版本}.{次版本}.{修订}
```

| 类型 | 含义 | 示例 |
|------|------|------|
| 主版本 | 重大变更，不兼容 | 2.0.0 |
| 次版本 | 新增功能，向后兼容 | 1.1.0 |
| 修订 | 错误修正 | 1.0.1 |

### 版本记录

```
文档版本: v1.0.0
创建日期: 2026-02-23
创建人: ceo-coach
变更内容: 初始版本
```
