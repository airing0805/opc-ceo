# 调度系统登录问题解决方案

## 问题描述

OPCCoachPlanning 调度任务以 SYSTEM 账户运行，无法访问用户的 Claude Code 登录状态。

**错误日志**：`Not logged in · Please run /login`

## 已尝试的解决方案

| 方案 | 状态 | 说明 |
|------|------|------|
| SYSTEM 账户 | 阻塞 | 无法访问用户登录状态 |
| Administrator 交互式账户 | 失败 | 错误代码 267009，需要密码验证 |

## 可行的解决方案

### 方案 A：配置 API Key（推荐）

1. 获取 Anthropic API Key
2. 配置环境变量或直接在脚本中设置

**配置方式 1**：在系统环境变量中设置
```
变量名: ANTHROPIC_API_KEY
值: your-api-key-here
```

**配置方式 2**：修改 bat 脚本直接设置（不推荐，仅用于测试）

### 方案 B：使用用户账户（需密码）

1. 提供 Administrator 账户密码
2. 修改注册脚本使用 `-LogonType Password`
3. 确保用户保持登录状态
