# OPC-CEO 设计意图

## 核心愿景

构建一个**一人公司的 CEO Agent 系统**，通过 Claude Code 管理个人电脑上的所有事务。

---

## 设计原则

### 1. 纯 Skills/Agents 架构

**意图**：不写 Python/JS 后端代码，完全基于 Claude Code 的 Skills 和 Agents 系统实现所有功能。

**原因**：
- 降低维护成本，无需管理复杂的软件栈
- 充分利用 Claude Code 的原生能力
- 通过自然语言即可扩展功能
- 避免过度工程化

### 2. 统一入口，角色分工

**意图**：所有操作通过 Claude Code 访问当前项目目录，由 CEO 角色协调分发到各个专业角色。

**原因**：
- 单一入口简化用户交互
- 专业角色各司其职，职责清晰
- 便于扩展新的专业角色

### 3. MCP Memory 作为数据层

**意图**：使用 MCP Memory 知识图谱存储所有数据（任务、知识、财务、日程等）。

**原因**：
- 无需部署数据库
- 数据以实体和关系的形式自然存储
- 支持语义搜索和关联查询
- 与 Claude Code 深度集成

### 4. 可程序化执行

**意图**：通过 Claude SDK/API 实现自动化执行，支持定时任务和批量操作。

**原因**：
- 支持无人值守的自动化工作流
- 可以通过脚本触发复杂操作
- 与外部系统集成

---

## 目标用户

**一人公司创业者 / 独立开发者 / 自由职业者**

核心需求：
- 任务和项目管理
- 知识库管理
- 时间和日程管理
- 财务记录
- 文件系统管理
- 工作流自动化

## 技术决策

### 选择 Claude Code Skills 而非传统开发

| 方面 | 传统开发 | Skills 方案 |
|------|----------|-------------|
| 开发成本 | 高（需要完整后端） | 低（只需编写 MD 文件） |
| 维护成本 | 高（依赖管理、升级） | 低（无依赖） |
| 扩展性 | 需要代码修改 | 自然语言描述即可 |
| 数据存储 | 需要数据库 | MCP Memory |
| 部署 | 需要服务器 | 本地运行 |

### 数据存储方案

| 数据类型 | 存储方式 |
|----------|----------|
| 结构化数据 | MCP Memory 实体 |
| 关系数据 | MCP Memory 关系 |
| 文件数据 | 本地文件系统 |
| 知识图谱 | MCP Memory |

## 使用场景示例

### 场景 1：每日工作规划

```
用户：「帮我规划今天的工作」

CEO Agent：
1. 调用 task-manager 获取今日待办
2. 调用 schedule-manager 获取今日日程
3. 调用 finance-manager 检查待处理账单
4. 综合分析，生成优先级建议
```

### 场景 2：项目知识管理

```
用户：「记录这个项目的设计决策」

CEO Agent：
1. 调用 knowledge-manager 创建知识实体
2. 建立与项目的关联关系
3. 可通过语义搜索检索
```

### 场景 3：自动化周报

```
用户：「每周五下午生成周报」

automation-manager：
1. 定义定时工作流
2. 周五触发：
   - task-manager 统计本周完成的任务
   - finance-manager 统计本周收支
   - 生成报告并保存
```
