# 快速参考

## 常用命令速查

### 决策支持

| 命令 | 说明 |
|------|------|
| 分析优先级 | 根据紧急度和重要性判断任务优先级 |
| 分配资源 | 根据任务需求分配执行角色和时间资源 |
| 记录决策 | 记录重要决策到知识图谱 |

### 上下文管理

| 命令 | 说明 |
|------|------|
| 保存上下文 | 保存当前会话的上下文状态 |
| 恢复上下文 | 从知识图谱恢复之前的上下文 |
| 查询历史 | 查询历史上下文记录 |

### 协调流程

| 命令 | 说明 |
|------|------|
| 分配单角色任务 | 分配给单一角色的简单任务 |
| 协调多角色任务 | 多角色协作的复杂任务 |
| 并行协调任务 | 多个角色同时执行独立任务 |

### 角色成长管理

| 命令 | 说明 |
|------|------|
| 监控角色表现 | 定期检查各角色的执行表现 |
| 触发能力提升 | 触发角色能力提升流程 |
| 更新能力版本 | 更新角色的能力版本号 |

### 角色创建

| 定义角色能力 | 定义新角色的核心能力 |
| 生成角色文档 | 生成角色设计文档和技能文件 |

### 角色修改

| 命令 | 说明 |
|------|------|
| 分析修改需求 | 分析角色修改的必要性 |
| 修改角色能力 | 修改角色的能力定义 |
| 更新版本号 | 更新角色的版本号 |

### 能力边界

| 命令 | 说明 |
|------|------|

| 查询角色边界 | 查询角色的能力边界 |

| 命令 | 说明 |

## 角色能力边界速查表

| 角色 | 核心能力 | 不能做 |
|------|----------|--------|
| task-manager | 任务CRUD、状态跟踪、分类统计 | 文件操作、知识推理、财务计算 |
| file-manager | 文件搜索、操作、组织、备份 | 任务状态管理、知识图谱操作 |
| knowledge-manager | 知识存储、检索、图谱构建 | 文件系统操作、财务记录 |
| finance-manager | 收入支出记录、分类、报告 | 文件操作、任务管理、知识推理 |
| schedule-manager | 事件创建、查询、时间块管理 | 文件操作、财务计算、知识存储 |
| automation-manager | 定时任务、工作流设计、触发器 | 直接执行业务逻辑 |
| claude-sdk-executor | API调用、批量处理、技能调用 | 业务决策、用户交互 |

## 任务类型-角色映射

| 任务类型 | 首选角色 |
|----------|----------|
| 创建任务 | task-manager |
| 查找文件 | file-manager |
| 记录支出 | finance-manager |
| 安排会议 | schedule-manager |
| 设置定时任务 | automation-manager |
| 批量处理 | claude-sdk-executor |
| 跨角色协调 | CEO |

## 优先级判断矩阵

```
┌─────────┬─────────┬─────────┬─────────┐
│         │ 重要性高 │ 重要性中 │ 重要性低 │
├─────────┼─────────┼─────────┼─────────┤
│ 紧急度高 │   P0    │   P1    │   P2    │
│ 紧急度中 │   P1    │   P2    │   P3    │
│ 紧急度低 │   P2    │   P3    │   P4    │
└─────────┴─────────┴─────────┴─────────┘
```

## 实体ID格式

| 类型 | 格式 | 示例 |
|------|------|------|
| 任务 | `TASK-YYYY-MM-DD-NNN` | `TASK-2026-02-23-001` |
| 项目 | `PROJ-NNN` | `PROJ-001` |
| 交易 | `TXN-YYYY-MM-DD-NNN` | `TXN-2026-02-23-001` |
| 事件 | `EVENT-YYYY-MM-DD-NNN` | `EVENT-2026-02-24-001` |
| 决策 | `DECISION-YYYY-MM-DD-NNN` | `DECISION-2026-02-23-001` |

## 红线清单

1. 连续3次重大决策失误
2. 2周内无可见成长
3. 隐瞒问题或虚假汇报
4. 推卸责任给下属角色
5. 无法有效协调团队资源
