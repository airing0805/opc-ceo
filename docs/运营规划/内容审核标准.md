# 内容审核标准

> 版本：v10.2.1  
> 创建日期：2026-02-26  
> 适用范围：所有内容生产与发布流程

## 概述

本文档定义内容审核的标准和流程，确保发布的内容符合质量要求、平台规范和法律法规。所有内容在发布前必须通过以下五个维度的审核。

---

## 1. 内容完整性

### 1.1 检查项

| 序号 | 检查项 | 说明 |
|------|--------|------|
| 1.1.1 | 标题完整性 | 标题是否存在、是否清晰表达主题 |
| 1.1.2 | 摘要/引言 | 是否有简短的摘要或引言引入正文 |
| 1.1.3 | 正文结构 | 正文是否包含必要的背景、论述、结论 |
| 1.1.4 | 结尾总结 | 是否有结尾总结或行动呼吁 |
| 1.1.5 | 内容连贯性 | 各段落之间逻辑是否连贯 |

### 1.2 合格标准

- 标题必须完整，能够清晰表达文章主题
- 正文必须包含引入、主体、结论三个基本部分
- 内容长度应符合平台要求（知乎≥800字，简书≥500字）
- 结尾必须包含总结或引导性语句

### 1.3 常见问题示例

| 问题类型 | 示例 | 纠正方式 |
|----------|------|----------|
| 标题缺失 | 无标题或标题为"无题" | 确保标题准确描述内容主题 |
| 虎头蛇尾 | 文章有精彩开头但无结尾 | 添加总结性结尾段落 |
| 内容不完整 | 只列出要点但未展开论述 | 补充完整的论述内容 |
| 结构混乱 | 跳跃性太大，缺乏过渡 | 添加过渡段落或连接词 |

## 2. 格式规范

### 2.1 检查项

| 序号 | 检查项 | 说明 |
|------|--------|------|
| 2.1.1 | 段落划分 | 段落是否清晰，长度是否适中 |
| 2.1.2 | 标题层级 | 标题层级是否规范（H1-H6） |
| 2.1.3 | 代码块 | 代码是否使用代码块格式 |
| 2.1.4 | 列表格式 | 列表是否使用统一格式 |
| 2.1.5 | 标点符号 | 标点使用是否规范 |
| 2.1.6 | 空格规范 | 中英文之间是否有空格 |
| 2.1.7 | 引用格式 | 引用内容是否使用正确格式 |

### 2.2 合格标准

- 段落长度适中，每段不超过200字
- 标题层级清晰，最多使用3级标题
- 代码必须使用代码块包裹，标注语言类型
- 列表使用统一的符号（- 或 1.）
- 中文使用全角标点，英文使用半角标点
- 中英文混排时，英文单词两侧有空格

### 2.3 常见问题示例

| 问题类型 | 示例 | 纠正方式 |
|----------|------|----------|
| 标题层级混乱 | 跳过H2直接使用H3 | 按顺序使用标题层级 |
| 代码未格式化 | `function test()` 放在正文中 | 使用代码块包裹 |
| 段落过长 | 单段落超过500字 | 拆分为多个段落 |
| 标点混用 | 中文中使用英文逗号 | 统一使用中文标点 |
| 缺少空格 | "这是一个测试"应为"这是一个 测试" | 添加适当空格 |

## 3. 敏感信息

### 3.1 检查项

| 序号 | 检查项 | 说明 |
|------|--------|------|
| 3.1.1 | 个人身份信息 | 是否包含真实姓名、身份证号、地址 |
| 3.1.2 | 联系方式 | 是否包含手机号、邮箱、家庭住址 |
| 3.1.3 | 账户凭证 | 是否包含密码、API Key、Token |
| 3.1.4 | 金融信息 | 是否包含银行卡号、支付账户 |
| 3.1.5 | 隐私内容 | 是否包含他人隐私信息 |

### 3.2 合格标准

- 禁止包含任何个人身份信息（姓名、身份证号）
- 禁止包含任何联系方式（手机号、邮箱需脱敏）
- 禁止包含任何账户凭证（密码、API Key、Token）
- 禁止包含金融信息（银行卡号、支付账户）
- 如需示例，使用虚构信息或占位符

### 3.3 常见问题示例

| 问题类型 | 示例 | 纠正方式 |
|----------|------|----------|
| 泄露手机号 | "如有疑问请联系：138-xxxx-xxxx" | 使用脱敏格式或删除 |
| 暴露密码 | password: "123456" | 删除或使用占位符 |
| API Key泄露 | api_key: "sk-xxxxx" | 使用环境变量或占位符 |
| 真实姓名 | "我的朋友张三" | 使用化名 |
| 他人隐私 | 未经同意提及他人信息 | 删除或匿名化处理 |

## 4. 平台合规

### 4.1 检查项

| 序号 | 检查项 | 说明 |
|------|--------|------|
| 4.1.1 | 广告内容 | 是否包含硬广告或软广告 |
| 4.1.2 | 引流信息 | 是否包含外链或联系方式引流 |
| 4.1.3 | 敏感词 | 是否包含平台敏感词 |
| 4.1.4 | 违规内容 | 是否包含违法、违规内容 |
| 4.1.5 | 版权内容 | 是否引用版权素材 |
| 4.1.6 | 抄袭内容 | 是否存在抄袭或洗稿 |

### 4.2 合格标准

- 禁止包含明显广告内容（产品推销、价格信息）
- 禁止包含引流信息（二维码、外部链接、诱导加好友）
- 禁止包含敏感词（需参考各平台敏感词库）
- 禁止包含违法违规内容
- 引用他人内容需注明来源
- 必须为原创内容，禁止抄袭或洗稿

### 4.3 常见问题示例

| 问题类型 | 示例 | 纠正方式 |
|----------|------|----------|
| 硬广告 | "现在购买仅需99元" | 改为客观描述 |
| 引流信息 | "想了解更多请加微信XXX" | 删除引流内容 |
| 敏感词 | 使用平台禁用词汇 | 替换为合规词汇 |
| 版权侵权 | 大段引用他人文章未注明 | 改为概括引用并注明来源 |
| 洗稿 | 简单改写他人文章 | 重新创作原创内容 |

## 5. 内容质量

### 5.1 检查项

| 序号 | 检查项 | 说明 |
|------|--------|------|
| 5.1.1 | 逻辑清晰 | 论证逻辑是否严密 |
| 5.1.2 | 错别字 | 是否有错别字、别字 |
| 5.1.3 | 语法正确 | 句子是否通顺 |
| 5.1.4 | 事实准确 | 引用的数据、事实是否准确 |
| 5.1.5 | 表达专业 | 用语是否专业、准确 |
| 5.1.6 | 价值提供 | 是否为读者提供价值 |

### 5.2 合格标准

- 文章逻辑清晰，论证有据可依
- 无错别字、别字（使用拼写检查工具）
- 语法正确，句子通顺
- 引用的数据和事实必须准确
- 使用专业、准确的术语
- 内容必须为读者提供实际价值

### 5.3 常见问题示例

| 问题类型 | 示例 | 纠正方式 |
|----------|------|----------|
| 错别字 | "的地得"使用错误 | 使用拼写检查工具校对 |
| 逻辑跳跃 | "因为A所以C"缺少B的论证 | 补充论证过程 |
| 事实错误 | "2023年发布"应为"2024年" | 核实并更正 |
| 用语不专业 | "这个很厉害" | 改为具体专业描述 |
| 缺乏价值 | 通篇废话无实质内容 | 增加有价值的内容 |

## 审核流程

### 5.1 审核责任人

| 内容类型 | 审核人 |
|----------|--------|
| 技术文章 | 技术审核员 |
| 运营文案 | 运营审核员 |
| 战略规划 | CEO/Coach |

### 5.2 审核步骤

1. **自检**：作者完成内容后进行自我检查
2. **交叉检查**：由其他角色进行复核
3. **最终审核**：由审核负责人进行最终确认
4. **发布**：审核通过后方可发布

### 5.3 审核记录

所有审核记录需保存，包括：
- 审核时间
- 审核人
- 发现的问题
- 整改情况
