# 时间块

## 概述

定义时间块划分策略，帮助合理安排一天的时间，提高工作效率。

## 时间块类型

### TB-DEEP (深度工作)

**说明**：需要高度专注、不被打扰的工作时间

**建议时长**：2-4 小时

**适合任务**：
- 核心功能开发
- 复杂问题调试
- 架构设计

**规则**：
- 关闭通知
- 专注编码/思考
- 不处理邮件/消息

### TB-MEET (会议沟通)

**说明**：与他人的沟通、讨论、协作

**建议时长**：1-2 小时

**适合任务**：
- 项目会议
- 需求讨论
- 代码审查

**规则**：
- 提前准备
- 控制时长
- 记录要点

### TB-ADMIN (行政事务)

**说明**：低优先级但必要的事务性工作

**建议时长**：0.5-1 小时

**适合任务**：
- 邮件处理
- 文档整理
- 行政事项

**规则**：
- 集中处理
- 快速决策
- 避免拖延

### TB-LEARN (学习成长)

**说明**：知识学习和技能提升

**建议时长**：1-2 小时

**适合任务**：
- 技术阅读
- 课程学习
- 技术调研

**规则**：
- 专注学习
- 做笔记
- 实践验证

### TB-BREAK (休息放松)

**说明**：恢复精力，避免过度疲劳

**建议时长**：0.5 小时

**适合活动**：
- 散步
- 运动
- 娱乐

**规则**：
- 完全放松
- 不思考工作
- 为下个时间块充电

## 时间块规划原则

### 原则 1：黄金时间段用于深度工作

```
将精力最旺盛的时间段安排 TB-DEEP
- 上午 9:00 - 12:00
- 下午 14:00 - 16:00
```

### 原则 2：深度工作连续化

```
避免频繁切换上下文：
┌─────────────────────────┐
│  TB-DEEP  (2-4小时)   │
└─────────────────────────┘

而非：
┌─────┬─────┬─────┬─────┐
│ DEEP │ ADMIN│ DEEP │ MEET│
│ 1h   │ 0.5h │ 1h   │ 1h  │
└─────┴─────┴─────┴─────┘
```

### 原则 3：会议集中安排

```
将会议集中在特定时间段
- 上午 10:00 - 11:00
- 下午 15:00 - 16:00

保留其他时间段用于深度工作
```

### 原则 4：定期休息

```
每 90-120 分钟深度工作后安排休息：
┌───────────────┬─────────┐
│   TB-DEEP     │ TB-BREAK│
│   (90-120分钟) │ (30分钟) │
└───────────────┴─────────┘
```

## 时间块冲突处理

### 冲突规则

| 冲突类型 | 处理方式 |
|---------|---------|
| TB-DEEP 与 TB-MEET | TB-MEET 优先（可协商） |
| TB-DEEP 与 TB-DEEP | 不能重叠 |
| TB-ADMIN 与 TB-DEEP | TB-DEEP 优先 |
| TB-BREAK 与 其他 | 优先休息 |

### 冲突解决

```
1. 识别冲突的时间块
2. 评估优先级
3. 调整时间或拒绝低优先级
4. 更新日程
```

```yaml
  - type: TB-DEEP

  - startTime: 2026-02-24 09:00
  - endTime: 2026-02-24 12:00
  - duration: 180
  - taskId: TASK-2026-02-24-001
  - status: completed
  - actualDuration: 150
  - createdAt: 2026-02-24T08:00:00Z
```

## 注意事项

- 深度工作需要连续时间块
- 定期休息提高效率
- 灵活调整计划
- 记录实际使用情况以优化
