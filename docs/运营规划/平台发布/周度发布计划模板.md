## 一、计划模板说明

### 1.1 用途

- 规划每周内容发布排期
- 合理分配平台发布时间
- 控制发布频率避免限流
- 追踪发布完成情况

### 1.2 使用方法

1. 每周初复制本模板创建新周计划
2. 填写本周待发布内容
3. 按平台分配发布时间
4. 每日更新完成状态
5. 周末进行复盘总结

### 1.3 发布原则

| 原则 | 说明 |
|------|------|
| 频率控制 | 每日2-3篇，避免限流 |
| 平台分散 | 不同平台交叉发布 |
| 时段优化 | 在最佳时段发布 |
| 间隔合理 | 同平台间隔>2小时 |

## 四、发布时间安排指南

### 4.1 最佳时段参考

| 时段 | 时间范围 | 适用平台 | 适用内容 |
|------|----------|----------|----------|
| 早高峰 | 08:00-09:30 | CSDN、知乎 | 技术教程、职场类 |
| 午休 | 12:00-14:00 | 简书、掘金 | 轻松类、工具推荐 |
| 晚高峰 | 18:00-21:00 | 全平台 | 通用（黄金时段） |
| 深夜 | 22:00-24:00 | 知乎 | 深度、情感类 |

### 4.2 时间安排原则

```
同一篇文章：
├── 首发平台：选择流量最大的平台
├── 时间：选择最佳时段
└── 后续：间隔24小时后发其他平台

同日多篇文章：
├── 间隔：至少2小时
├── 平台：交叉不同平台
└── 时段：覆盖不同高峰期
```

### 4.3 时间分配示例

**工作日（发布2-3篇）**：

```
08:30 │ CSDN发布（技术教程）
      │
12:00 │ 简书发布（工具推荐）
      │
19:00 │ 知乎发布（深度分析）
```

**周末（发布1-2篇）**：

```
10:00 │ 掘金发布（技术分享）
      │
15:00 │ 数据采集与复盘
```
