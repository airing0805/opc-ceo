# CSDN文章发布模板

> 本文档为CSDN平台内容创作提供标准化模板，适用于技术教程、工具推荐、实战案例等类型文章。

## 模板风格定位

**深度干货风格**：以解决实际问题为导向，提供可复制的技术方案和代码示例。

| 维度 | 要求 |
|------|------|
| 字数 | 800-1500字（不含代码） |
| 代码占比 | 30%-50% |
| 阅读时间 | 5-8分钟 |
| 目标读者 | 开发者、技术爱好者 |

---

## 一、标题格式规范

### 标题公式

```
[数字/形容词] + [核心关键词] + [利益点/效果]
```

### 标题类型对照表

| 类型 | 公式 | 示例 |
|------|------|------|
| **数字型** | X个方法/X分钟/X步 | 《5分钟搭建本地AI开发环境》 |
| **对比型** | A vs B / 从A到B | 《从Git小白到团队协作：一份完整指南》 |
| **痛点型** | 解决XX问题 / XX不再XX | 《还在手动部署？这个工具让你早点下班》 |
| **揭秘型** | 揭秘XX / XX背后的原理 | 《揭秘：大厂都在用的代码审查工具》 |
| **教程型** | XX教程 / 手把手教你 | 《手把手教你：用Claude Code构建Agent工作流》 |

### 标题禁忌

- 禁止标题党（内容与标题严重不符）
- 禁止过度夸张（"震惊"、"必看"等）
- 禁止敏感词（涉政、涉黄、涉暴）
- 标题长度控制在20-30字

## 三、正文结构

### 标准结构：问题-方案-代码-总结

```
1. 背景介绍（100-150字）
   - 为什么需要这个技术/工具
   - 当前方案的痛点

2. 核心方案（200-300字）
   - 整体思路
   - 架构图/流程图（可选）

3. 实现步骤（300-500字 + 代码）
   - 步骤1：环境准备
   - 步骤2：核心代码实现
   - 步骤3：测试验证

4. 进阶优化（100-200字）
   - 性能优化建议
   - 常见问题解答

5. 总结（100字）
   - 关键收获
   - 下一步行动
```

## 四、代码块格式规范

### 代码块要求

| 项目 | 规范 |
|------|------|
| 语言标识 | 必须标注（如 `python`、`bash`、`json`） |
| 注释 | 关键行必须添加注释 |
| 完整性 | 代码可独立运行（包含必要的import） |
| 长度 | 单块不超过50行，超长需拆分 |

### 代码块示例

````markdown
```python
# 导入必要的库
import json
from pathlib import Path

def load_config(config_path: str) -> dict:
    """
    加载配置文件
    
    Args:
        config_path: 配置文件路径
    Returns:
        配置字典
    """
    path = Path(config_path)
    
    # 检查文件是否存在
    if not path.exists():
        raise FileNotFoundError(f"配置文件不存在: {config_path}")
    
    # 读取并解析JSON
    with open(path, 'r', encoding='utf-8') as f:
        return json.load(f)

# 使用示例
config = load_config("config.json")
print(config)
```
````

### 代码注释规范

```python
# ✅ 好的注释：解释"为什么"
# 使用缓存避免重复计算，提升性能3倍
result = cache.get(key) or expensive_calculation()

# ❌ 坏的注释：重复"是什么"
# 设置x等于10
x = 10
```

## 六、标签和分类建议

### 标签选择原则

1. **主标签**（1-2个）：与文章核心主题直接相关
2. **次标签**（2-3个）：相关技术栈、应用场景
3. **总数量**：3-5个为宜

### 热门标签对照表

| 技术方向 | 推荐标签 |
|----------|----------|
| AI/LLM | 人工智能、AIGC、ChatGPT、Claude、大模型 |
| 前端 | 前端、Vue、React、JavaScript、TypeScript |
| 后端 | 后端、Python、Java、Spring、微服务 |
| 运维 | DevOps、Docker、Kubernetes、Linux、CI/CD |
| 工具 | 开发工具、效率提升、自动化 |

### 分类选择

- 技术教程 → 「原创」
- 工具推荐 → 「工具」或「经验分享」
- 实战案例 → 「项目实战」
- 学习笔记 → 「学习」
