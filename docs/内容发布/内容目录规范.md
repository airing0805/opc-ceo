# 内容目录规范

> 统一管理所有内容资产，明确素材与发布版本的区分
> 版本：V1.0 | 更新日期：2026-02-25

---

## 一、规范目的

- 区分内容素材与发布版本，避免混乱
- 统一命名规范，便于检索和管理
- 建立清晰的目录结构，支持多平台发布

---

## 二、目录结构

```
OPC-CEO/
├── content/                      # 内容素材库（原始稿件）
│   ├── README.md                 # 内容仓库说明
│   ├── 011-MCP是什么/            # 选题001
│   │   ├── README.md            # 内容元数据
│   │   ├── csdn/
│   │   │   └── 文章.md          # CSDN原始稿件
│   │   ├── xiaohongshu/
│   │   │   └── 文章.md          # 小红书原始稿件
│   │   └── xiaoyu/
│   │       └── 文章.md          # 闲鱼原始稿件
│   ├── 012-xxx/                 # 选题002
│   └── ...
│
└── docs/
    └── 内容发布/                  # 发布版本库（已发布/待发布）
        ├── README.md             # 发布仓库说明
        ├── 011-MCP是什么一篇文章讲清楚.md
        ├── 011-MCP是什么一篇文章讲清楚_CSDN.md
        ├── 011-MCP是什么一篇文章讲清楚_小红书.md
        ├── 2026年3月内容发布计划.md
        └── 发布追踪.md            # 追踪模板（新增）
```

## 三、目录职责

### 3.1 content/ - 内容素材库

| 属性 | 说明 |
|------|------|
| 定位 | 原始稿件存放地 |
| 状态 | 创作中、待适配、待发布 |
| 管理者 | 内容 Agent |
| 命名格式 | `{序号}-{选题名称}/` |

**使用场景**：
- 撰写新内容时在 content/ 下创建
- 同一内容多平台适配时，在平台子目录下保存
- 内容修改时优先修改 content/ 下的素材

### 3.2 docs/内容发布/ - 发布版本库

| 属性 | 说明 |
|------|------|
| 定位 | 已发布/待发布版本存放地 |
| 状态 | 待发布、已发布、已下架 |
| 管理者 | 运营 Agent |
| 命名格式 | `{序号}-{标题}_{平台}.md` |

**使用场景**：
- 内容发布后，将最终版本复制到 docs/内容发布/
- 记录各平台发布状态和链接
- 追踪发布数据

## 四、命名规范

### 4.1 内容文件夹 (content/)

```
{序号}-{选题名称}/
```

| 字段 | 规则 | 示例 |
|------|------|------|
| 序号 | 3位数字，从001开始 | 011 |
| 选题名称 | 中文，简洁明了 | MCP是什么 |
| 全称 | 用 - 连接 | 011-MCP是什么/ |

### 4.2 发布版本 (docs/内容发布/)

```
{序号}-{标题}_{平台}.md
```

| 字段 | 规则 | 示例 |
|------|------|------|
| 序号 | 与素材目录一致 | 011 |
| 标题 | 简洁，30字内 | MCP是什么一篇文章讲清楚 |
| 平台 | 英文小写 | _csdn, _xiaohongshu, _xiaoyu |
| 无平台后缀 | 综合版/原始版 | 011-MCP是什么一篇文章讲清楚.md |

### 4.3 平台标识

| 平台 | 标识 | 示例 |
|------|------|------|
| CSDN | _csdn | 011-MCP是什么一篇文章讲清楚_csdn.md |

## 五、状态管理

### 5.1 内容素材状态 (content/README.md)

| 状态 | 说明 | 标记 |
|------|------|------|
| 创作中 | 正在撰写中 | 🖊️ |

### 5.2 发布版本状态 (发布追踪.md)

| 状态 | 说明 | 标记 |
|------|------|------|
| 待发布 | 等待发布 | ⏳ |
| 已发布 | 已发布到平台 | ✅ |
| 已下架 | 从平台下架 | ❌ |
| 更新中 | 内容更新中 | 🔄 |

## 六、工作流程

### 6.1 内容创作流程

```
1. 选题确认
   ↓
2. 在 content/ 创建选题目录（如 011-MCP是什么/）
   ↓
3. 撰写原始稿件（csdn/文章.md）
   ↓
4. 适配各平台（xiaohongshu/文章.md 等）
   ↓
5. 更新 content/README.md 状态
```

### 6.2 内容发布流程

```
1. 确认 content/ 素材已准备完成
   ↓
2. 复制最终版本到 docs/内容发布/
   ↓
3. 在发布平台填写/上传内容
   ↓
4. 记录发布链接和状态到 发布追踪.md
   ↓
5. 更新 content/README.md 状态为"已发布"
```
