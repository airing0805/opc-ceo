# 角色成长管理

## 功能概述

监控各角色表现，触发能力进化流程，维护角色能力版本。

## 操作命令

### 命令 1：监控角色表现

**用途**：定期检查各角色的执行表现

**语法**：
```
监控角色表现:
- 角色: <角色名>
- 时间范围: <范围>
- 评估维度: [维度列表]
```

**参数**：

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| 角色 | string | 是 | 要监控的角色名 |
| 时间范围 | string | 否 | 如 "本周"、"本月" |
| 评估维度 | array | 否 | 完整性、准确性、效率等 |

**示例**：
```
监控角色表现:
- 角色: task-manager
- 时间范围: 本周
- 评估维度: [完整性, 准确性, 效率]
```

### 命令 2：触发能力提升

**用途**：当角色表现不佳时触发能力提升流程

**语法**：
```
触发能力提升:
- 角色: <角色名>
- 问题类型: <类型>
- 问题描述: <描述>
- 改进方案: <方案>
- 期望效果: <效果>
```

**参数**：

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| 角色 | string | 是 | 需要提升的角色 |
| 问题类型 | string | 是 | 能力不足/效率下降/边界模糊 |
| 问题描述 | string | 是 | 详细问题描述 |
| 改进方案 | string | 是 | 改进方案描述 |
| 期望效果 | string | 是 | 期望达到的效果 |

**示例**：
```
触发能力提升:
- 角色: task-manager
- 问题类型: 能力不足
- 问题描述: 无法识别任务间的依赖关系
- 改进方案: 添加依赖分析能力
- 期望效果: 能正确识别80%以上的依赖关系
```

### 命令 3：更新能力版本

**用途**：更新角色的能力版本号

**语法**：
```
更新能力版本:
- 角色: <角色名>
- 新版本: <版本号>
- 更新内容: [更新列表]
```

**参数**：

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| 角色 | string | 是 | 角色名 |
| 新版本 | string | 是 | 新版本号 |
| 更新内容 | array | 是 | 更新内容列表 |

**示例**：
```
更新能力版本:
- 角色: task-manager
- 新版本: 1.1.0
- 更新内容: [新增依赖分析, 新增智能优先级]
```

## 成长触发条件

| 触发条件 | 说明 |
|----------|------|
| 执行失败 | 角色无法完成分配的任务 |
| 效率下降 | 角色执行时间明显增加 |
| 用户反馈 | 用户对某角色表达不满 |
| 新需求 | 现有能力无法满足新场景 |
| 周期评估 | 定期（每周/每月）评估角色表现 |

## MCP Memory 使用

### 创建能力记录

```
使用 mcp__memory__create_entities:
{
  "name": "capability-task-manager",
  "entityType": "RoleCapability",
  "observations": [
    "roleName: task-manager",
    "version: 1.0.0",
    "updatedAt: 2026-02-23",
    "capabilities: [任务CRUD, 状态跟踪, 分类统计]",
    "performance: 85%",
    "improvements: []"
  ]
}
```

### 创建成长记录

```
使用 mcp__memory__create_entities:
{
  "name": "growth-task-manager-2026-02-23",
  "entityType": "GrowthRecord",
  "observations": [
    "role: task-manager",
    "trigger: 能力不足",
    "issue: 无法识别任务依赖",
    "solution: 添加依赖分析能力",
    "status: pending",
    "createdAt: 2026-02-23"
  ]
}
```

## 成长质量评估

| 维度 | 合格 | 优秀 | 卓越 |
|------|------|------|------|
| 响应速度 | 24小时内 | 12小时内 | 6小时内 |
| 改进效果 | 解决问题 | 显著改善 | 超出预期 |
| 影响范围 | 局部优化 | 角色提升 | 全局改进 |

## 错误处理

| 错误类型 | 处理方式 |
|----------|----------|
| 改进无效 | 分析原因，调整方案 |
| 副作用 | 回滚版本，重新设计 |
| 资源不足 | 调整优先级，分阶段实施 |
