### 2.1 掘金热门标题特征

| 特征 | 说明 | 示例 |
|------|------|------|
| 数字量化 | 具体数字吸引眼球 | "5分钟搞懂..."、"10个技巧..." |

## 前言

你在Vue2开发中是否遇到过这些问题：

- 组件越来越大，data/methods/computed分散各处
- 相同逻辑难以复用，mixin又容易冲突
- TypeScript支持不够友好

如果你也为此困扰，那么组合式API就是为你准备的。

**阅读本文你将收获：**

- 理解组合式API的设计理念
- 掌握setup函数的使用方法
- 学会逻辑复用的最佳实践

---

## 一、组合式API是什么？

### 1.1 定义

组合式API（Composition API）是Vue3引入的一种新的编写组件逻辑的方式。

> 核心思想：把相关逻辑组织在一起，而不是按选项类型分散。

### 1.2 为什么需要组合式API

| 问题 | Options API | Composition API |
|------|-------------|-----------------|
| 逻辑组织 | 按选项分散 | 按功能聚合 |
| 逻辑复用 | mixin有冲突风险 | 组合函数清晰 |
| 类型推导 | 支持有限 | 完美支持 |

---

## 二、核心概念

### 2.1 setup函数

setup是组合式API的入口：

```javascript
import { ref } from 'vue'

export default {
  setup() {
    const count = ref(0)
    
    const increment = () => {
      count.value++
    }
    
    return {
      count,
      increment
    }
  }
}
```

### 2.2 响应式API

| API | 用途 |
|-----|------|
| ref | 基础类型响应式 |
| reactive | 对象响应式 |
| computed | 计算属性 |
| watch | 侦听器 |
