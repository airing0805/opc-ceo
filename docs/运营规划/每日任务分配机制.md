## 一、机制概述

### 1.1 核心目标

建立一套标准化的每日任务分配机制，实现：

- **自动化任务领取**：CEO 每日启动时自动获取当日待办任务
- **优先级智能排序**：基于战略对齐度、紧急程度、可行性自动排序
- **状态实时追踪**：任务执行全程可追溯
- **完成标准明确**：每个任务有清晰的完成定义

### 1.2 角色分工

| 角色 | 职责 | 触发时机 |
|------|------|----------|
| **Coach** | 制定规范、规划任务、审核完成标准 | 规划阶段 |
| **CEO** | 执行任务、记录日志、汇报进度 | 执行阶段 |
| **用户** | 确认决策、协助操作、验收成果 | 关键节点 |

### 1.3 适用范围

本机制适用于所有运营相关的日常任务分配：

- 内容生产任务（文章撰写、视频制作）
- 账号运营任务（发布、互动、数据分析）
- 产品开发任务（课程、电子书、工具）
- 数据采集任务（指标追踪、报告生成）

## 三、任务优先级判断标准

### 3.1 任务价值公式

```
任务价值 = 战略对齐度 × 紧急程度 × 实现可行性
```

### 3.2 评分标准

#### 战略对齐度（0-1分）

| 分值 | 标准 | 示例 |
|------|------|------|
| 0.95-1.0 | 直接贡献Q2收入目标（≥1万） | 电子书发布、课程上线 |
| 0.85-0.94 | 支撑核心业务运转 | 内容发布、账号运营 |
| 0.70-0.84 | 提升运营效率 | 工具优化、流程改进 |
| 0.50-0.69 | 辅助性工作 | 文档整理、数据备份 |
| 0-0.49 | 与战略关联度低 | 探索性任务 |

#### 紧急程度（0-1分）

| 分值 | 标准 | 示例 |
|------|------|------|
| 0.90-1.0 | 3天内截止或阻塞核心任务 | 账号注册、平台确认 |
| 0.80-0.89 | 1周内截止 | 电子书排版、内容发布 |
| 0.60-0.79 | 2周内截止 | 模板制定、系统完善 |
| 0.40-0.59 | 本月内完成 | 长期规划任务 |
| 0-0.39 | 无明确截止时间 | 探索性学习 |

#### 实现可行性（0-1分）

| 分值 | 标准 | 示例 |
|------|------|------|
| 0.90-1.0 | 无依赖，CEO可直接执行 | 文档撰写、代码修改 |
| 0.80-0.89 | 有工具/模板支持，执行简单 | 数据采集、内容发布 |
| 0.60-0.79 | 需要一定准备或学习 | 新平台操作、复杂分析 |
| 0.40-0.59 | 需要外部资源或协助 | 账号注册、平台对接 |
| 0-0.39 | 依赖多个不确定因素 | 第三方合作、审批流程 |

### 3.3 优先级分类

| 任务价值 | 优先级 | 标识 | 处理原则 |
|----------|--------|------|----------|
| >0.80 | 最高 | P0-紧急 | 立即处理，当日完成 |
| 0.70-0.80 | 高 | P0 | 优先处理，当日完成 |
| 0.50-0.70 | 中 | P1 | 本周完成 |
| 0.30-0.50 | 低 | P2 | 本月完成 |
| <0.30 | 可延后 | P3 | 视情况处理 |

### 3.4 优先级调整规则

| 情况 | 调整规则 | 说明 |
|------|----------|------|
| 截止日期临近 | +0.1/天 | 距截止每少1天增加0.1 |
| 阻塞其他任务 | +0.2 | 有阻塞标记的任务优先 |
| 用户明确要求 | +0.3 | 用户指定优先执行 |
| 依赖任务完成 | 自动提升 | 依赖解除后重新评估 |

## 五、任务状态追踪方法

### 5.1 任务状态定义

| 状态 | 标识 | 定义 | 触发条件 |
|------|------|------|----------|
| 待分配 | 📋 | 任务已创建，等待分配 | 任务创建时 |
| 待执行 | ⏳ | 已分配到日期，等待开始 | 任务分配时 |
| 进行中 | 🔄 | 正在执行 | 任务开始时 |
| 已完成 | ✅ | 达到完成标准 | 完成确认时 |
| 阻塞 | 🚫 | 无法继续，等待外部条件 | 遇到阻塞时 |
| 取消 | ⏹️ | 任务不再需要 | 用户确认取消 |

### 5.2 追踪位置

任务状态在以下位置同步更新：

| 位置 | 更新时机 | 更新内容 |
|------|----------|----------|
| `docs/沟通文档/任务分配.md` | 任务状态变化时 | 任务状态标记 |
| `docs/执行日志/YYYY-MM-DD.md` | 每日结束时 | 执行记录 |
| `docs/运营规划/数据看板.md` | 每周汇总时 | 统计数据 |

### 5.3 任务状态追踪表

在 `任务分配.md` 中使用以下格式追踪：

```markdown
| 任务ID | 任务名称 | 优先级 | 状态 | 更新日期 |
|--------|----------|--------|------|----------|
| v5.2.1.1 | 整合电子书四章 | P0 | 🔄 进行中 | 2026-02-26 |
| v5.2.1.2 | 统一格式排版 | P0 | ⏳ 待执行 | 2026-02-26 |
| v5.2.1.3 | 添加目录页眉 | P1 | 📋 待分配 | - |
```

### 5.4 阻塞任务处理

当任务被阻塞时，记录阻塞信息：

```markdown

| 任务ID | 阻塞原因 | 阻塞时间 | 解除条件 | 责任人 |
```

## 六、任务完成标准

### 6.1 完成标准模板

每个任务必须定义明确的完成标准：

```markdown
### 任务: {任务名称}

**完成标准**:
- [ ] 标准1: {具体可验证的标准}
- [ ] 标准2: {具体可验证的标准}
- [ ] 标准3: {具体可验证的标准}

**交付物**:
- 交付物1: {文件路径或链接}
- 交付物2: {文件路径或链接}
```

### 6.2 任务类型完成标准

| 任务类型 | 完成标准 | 交付物 |
|----------|----------|--------|
| **文章撰写** | - 符合模板格式<br>- 字数达标<br>- 无错别字<br>- 标题吸引人 | Markdown文件 |
| **电子书** | - 内容完整<br>- 格式统一<br>- 排版美观<br>- PDF生成 | MD + PDF文件 |
| **账号注册** | - 账号创建成功<br>- 基础信息填写完成<br>- 可正常登录 | 账号信息记录 |
| **产品上架** | - 产品创建完成<br>- 描述/定价填写<br>- 可正常购买 | 产品链接 |
| **数据采集** | - 数据已收集<br>- 已录入模板<br>- 无遗漏 | 数据文件 |
| **代码开发** | - 功能实现<br>- 测试通过<br>- 文档更新 | 代码 + 文档 |

### 6.3 完成确认流程

```
任务执行完成
    │
    ▼
CEO 自查（对照完成标准）
    │
    ├─ 不符合 → 继续执行
    │
    ▼ 符合
更新任务状态为"已完成"
    │
    ▼
记录执行日志
    │
    ▼
向用户汇报完成情况
    │
    ▼
用户确认（如有需要）
```

### 6.4 完成质量检查清单

CEO 在标记任务完成前必须检查：

- [ ] 所有完成标准项均已满足
- [ ] 交付物已生成并保存到正确位置
- [ ] 无遗留问题或已记录待处理
- [ ] 相关文档已更新
- [ ] 执行日志已记录

## 八、自动化建议

### 8.1 可自动化分配的任务

| 任务类型 | 自动化程度 | 自动化方式 | 说明 |
|----------|------------|------------|------|
| **选题库任务** | 高 | 按选题库优先级自动提取 | 基于选题库评分自动排序 |
| **固定周期任务** | 高 | 定时任务自动添加 | 如每日数据采集、周报生成 |
| **依赖触发任务** | 中 | 依赖任务完成后自动激活 | 监控前置任务状态 |
| **数据采集任务** | 高 | Claude Code 自动执行 | 读取数据并录入模板 |

**自动化任务清单**:

```
可自动化任务示例:
1. 每日数据采集（早9点自动执行）
2. 每周执行日志汇总（周日自动执行）
3. 选题库任务提取（按优先级自动排序）
4. 依赖任务状态监控（实时检查）
5. 任务完成提醒（自动更新状态）
```

### 8.2 需要人工判断的任务

| 任务类型 | 判断要点 | 建议处理方式 |
|----------|----------|--------------|
| **战略决策** | 需要用户确认方向 | 汇报选项，等待用户决策 |
| **内容创作** | 需要创意和判断 | AI辅助生成，人工审核 |
| **问题处理** | 需要分析根因 | 提供建议，等待确认 |
| **外部协作** | 涉及第三方 | 用户主导，CEO辅助 |
| **质量审核** | 需要主观判断 | 提供检查清单，用户最终确认 |

**人工判断任务示例**:

```
需要人工判断的任务:
1. 上架平台选择（小鹅通 vs 有赞）- 需用户决策
2. 内容定位调整 - 需用户确认
3. 账号注册操作 - 需用户提供验证码
4. 产品定价策略 - 需用户确认
5. 内容质量审核 - 需用户最终确认
```

### 8.3 Claude Code 任务追踪方案

#### 方案一：启动时自动汇报

在 CEO 会话启动时，自动执行以下流程：

```
1. 读取 docs/沟通文档/任务分配.md
2. 筛选状态为"待执行"的任务
3. 计算任务价值并排序
4. 生成今日任务清单
5. 向用户汇报并等待确认
```

#### 方案二：执行中状态追踪

在任务执行过程中，自动记录：

```
1. 任务开始时间
2. 关键执行步骤
3. 遇到的问题
4. 任务结束时间
5. 执行结果
```

#### 方案三：日终自动总结

在会话结束时，自动生成：

```
1. 今日完成任务列表
2. 任务执行统计
3. 遇到的问题汇总
4. 明日任务预览
5. 待用户确认事项
```
