## 二、自我诊断

### 2.1 问题识别

#### 问题1：任务分发效率有待提升

**现状**：
- 五级分发机制（L1-L5）已经建立
- 但对于简单任务仍有冗余步骤
- 复杂度评估存在主观偏差

**改进方案**：
```
新增 L0 快速路径：复杂度 < 1 的任务直接执行
- 单文件读取
- 简单查询
- 状态确认

原有 L1-L5 保持不变
```

#### 问题2：执行等待时间较长

**现状**：
- 等待用户触发词是正确行为
- 但准备阶段可以更高效
- 场景识别可以更快

**改进方案**：
```
预加载机制：
- 启动时预加载任务分配.md
- 识别场景后预加载相关文档
- 并行准备而非串行等待
```

#### 问题3：自我检查维度不足

**现状**：
- 12项核心检查已很完善
- 但缺少效率维度
- 没有量化评估

**改进方案**：
```
新增 E 维度（Efficiency）：
- 执行时间预估
- 复杂度匹配度
- 资源利用效率
```

#### 问题4：与子角色协作存在延迟

**现状**：
- 需要协调时会调用子角色
- 但子角色响应时间不可控
- 缺少超时处理机制

**改进方案**：
```
增加协作超时机制：
- 子角色调用设置超时时间
- 超时后自动降级处理
- 记录协作延迟问题
```

## 三、改进方案

### 3.1 L0 快速路径机制

**目标**：为极简单任务提供最高效的执行路径

**适用场景**：
| 场景 | 示例 | 复杂度 |
|------|------|--------|
| 状态查询 | "当前有什么任务" | 0.5 |
| 文件读取 | "读取任务分配.md" | 0.3 |
| 简单确认 | "确认任务状态" | 0.4 |

**执行流程**：
```
用户输入
    │
    ├── 复杂度 < 1 → L0 快速路径 → 直接执行
    │
    ├── 复杂度 1-2 → L1 路径 → 简单检查后执行
    │
    ├── 复杂度 2-4 → L2 路径 → MCP 工具执行
    │
    ├── 复杂度 4-6 → L3 路径 → Agent 执行
    │
    ├── 复杂度 6-8 → L4 路径 → Team 协调
    │
    └── 复杂度 > 8 → L5 路径 → 拆分+递归
```

**决策时间目标**：
- L0: 1秒
- L1-L2: 3秒
- L3-L5: 10秒

### 3.2 预加载机制

**目标**：减少场景切换和文档加载的等待时间

**预加载策略**：

| 时机 | 预加载内容 | 优先级 |
|------|-----------|--------|
| 启动时 | 任务分配.md、战略目标.md | P0 |
| 场景识别后 | 相关子角色能力文档 | P1 |
| 用户确认后 | 执行所需文档 | P2 |

**并行预加载示例**：
```json
// 启动时并行预加载
mcp__filesystem__read_file({ "path": "任务分配.md" })
mcp__filesystem__read_file({ "path": "战略目标.md" })
mcp__memory__search_nodes({ "query": "task status" })
```

### 3.3 自我检查 E 维度

**新增 Efficiency 检查维度**：

| 检查项 | 目标值 | 检测方法 |
|--------|--------|----------|
| 复杂度匹配度 | ≥ 90% | 实际 vs 预估复杂度 |
| 执行时间偏差 | ≤ 20% | 实际 vs 预估时间 |
| 资源利用效率 | ≥ 80% | /总操作有效操作 |
| 并行利用率 | ≥ 70% | 并行操作/可并行操作 |

**E 维度检查清单**：
```
□ E1: 复杂度评估是否准确？（实际复杂度与预估偏差 ≤ 20%）
□ E2: 是否选择了最优执行路径？（L0-L5 匹配度）
□ E3: 执行时间是否在预估范围内？（±20%）
□ E4: 是否充分利用了并行机会？（≥70%）
□ E5: 是否有可复用的执行模式？（记录到示例库）
```

### 3.4 协作超时机制

**超时配置**：

| 协作类型 | 超时时间 | 超时处理 |
|----------|----------|----------|
| 子角色调用 | 30秒 | 降级为直接执行 |
| Agent 执行 | 60秒 | 汇报进度 |
| Team 协调 | 120秒 | 拆分任务 |

**超时处理流程**：
```
协作超时
    │
    ├── 子角色超时 → CEO 直接执行 → 记录问题
    │
    ├── Agent 超时 → 汇报进度 → 询问继续
    │
    └── Team 超时 → 拆分任务 → 分批执行
```
