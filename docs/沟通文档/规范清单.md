## 五、文档格式规范

### 任务表格格式
```
| 任务 ID | 标题 | 状态 | 说明 |
|---------|------|------|------|
| v2.3.1.1 | 任务标题 | ✅ 完成 | 说明内容 |
```

### 状态标记
| 状态 | 符号 |
|------|------|
| 待执行 | `[ ]` |
| 进行中 | `[🚧]` |
| 已完成 | `[x]` 或 ✅ |
| 部分完成 | ⚠️ |
| 暂停 | ⏸️ |
| 阻塞 | `[🚫]` |
| 废弃 | `[❌]` |

### 规范检查点
- [ ] 表格有正确的列对齐
- [ ] 使用标准状态标记符号
- [ ] 任务标题格式统一

## 六、沟通流程规范

### 任务分配流程
1. 分析用户意图
2. 确定负责人角色
3. 创建任务记录（任务分配.md）
4. 通过 Task 工具调用角色
5. 等待角色执行
6. 接收角色汇报（汇报总结.md）
7. 验收结果 → 更新任务状态

### 沟通文档结构
```
docs/沟通文档/
├── 文档沟通规范.md
├── 任务分配.md
├── 汇报总结.md
├── 协调记录.md
├── CEO-用户沟通.md
├── 角色记忆更新.md
└── 进化决策.md
```

### 规范检查点
- [ ] 目录结构完整
- [ ] 文档模板正确
- [ ] 沟通流程被遵守

## 七、冲突检测规则

### 命名冲突处理
| 原则 | 说明 |
|------|------|
| 时间优先 | 后定义的规范优先于先定义的 |
| 覆盖原则 | 主规范覆盖子规范 |
| 明确优先 | 有明确示例的规范优先 |
| 文档记录 | 冲突解决必须记录到变更历史 |

### 冲突类型
| 冲突类型 | 检测方法 |
|----------|----------|
| 命名冲突 | 对比不同文档中的ID格式定义 |
| 状态冲突 | 检查同一任务的状态标记是否一致 |
| 格式冲突 | 对比任务分配.md与规范模板 |
| 层级冲突 | 检查版本号层级是否一致 |
