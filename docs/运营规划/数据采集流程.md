## 一、流程概述

本文档定义每日运营数据的采集流程，确保数据采集的完整性、准确性和时效性。采集的数据将用于更新 [数据看板](数据看板.md) 和支持运营决策。

### 核心原则

1. **每日采集**：固定时间采集，确保数据连续性
2. **统一格式**：使用标准化模板记录，便于分析
3. **自动优先**：能自动采集的尽量自动化，减少人工错误
4. **多重验证**：关键数据需交叉验证，确保准确

## 三、各平台数据采集方法

### 3.1 公众号（微信公众平台）

**采集入口**：https://mp.weixin.qq.com

#### 数据位置

| 数据项 | 路径 | 说明 |
|--------|------|------|
| 单篇数据 | 内容管理 - 已发布 - 数据分析 | 每篇文章的阅读、点赞、在看 |
| 粉丝数据 | 用户管理 - 用户分析 | 新增关注、取消关注、净增 |
| 汇总数据 | 统计 - 图文分析 | 按日期查看汇总 |

#### 采集步骤

1. 登录微信公众平台
2. 进入「统计」→「图文分析」
3. 选择日期范围（前一日）
4. 记录以下数据：
   - 总阅读量
   - 总点赞量
   - 总在看量
   - 分享次数
   - 新增关注数

#### 数据格式

```markdown
| 日期 | 文章标题 | 阅读量 | 点赞 | 在看 | 分享 | 新增关注 |
|------|----------|--------|------|------|------|----------|
| YYYY-MM-DD | {标题} | {num} | {num} | {num} | {num} | {num} |
```

## 四、数据记录时间点

### 4.1 每日采集时间表

| 时间段 | 采集内容 | 耗时预估 | 责任人 |
|--------|----------|----------|--------|
| 09:00 - 09:15 | 公众号前日数据 | 5分钟 | CEO |
| 09:15 - 09:30 | CSDN前日数据 | 5分钟 | CEO |
| 09:30 - 09:45 | 小红书前日数据 | 5分钟 | CEO |
| 09:45 - 10:00 | 抖音前日数据 | 5分钟 | CEO |

| 21:15 - 21:30 | 数据汇总与看板更新 | 10分钟 | CEO |

### 4.2 采集时间原则

1. **固定时间**：每天在固定时间采集，减少遗忘
2. **错峰采集**：避开平台数据更新高峰期
3. **缓冲时间**：预留足够时间处理异常情况
4. **即时记录**：采集后立即记录，避免数据丢失

## 六、数据分析周期

### 6.1 分析周期定义

| 周期 | 频率 | 内容 | 输出物 |
|------|------|------|--------|
| 日分析 | 每日 | 前日数据概览、异常预警 | 日数据记录 |
| 周分析 | 每周一 | 上周数据汇总、趋势分析 | 周报 + 看板更新 |
| 月分析 | 每月1日 | 上月数据汇总、目标完成度 | 月报 + 策略调整建议 |
| 季分析 | 季度首日 | 上季度复盘、下季度规划 | 季报 + 目标修订 |

### 6.2 日分析流程

```
09:00 采集数据
  ↓
09:30 数据录入
  ↓
09:45 异常检测（日环比 > 50% 触发预警）
  ↓
10:00 更新看板（如有异常，标记并备注）
```

### 6.3 周分析流程

```
周一 09:00 汇总上周数据
  ↓
周一 10:00 计算周环比
  ↓
周一 11:00 更新看板
  ↓
周一 14:00 生成周报
  ↓
周一 15:00 复盘会议（如有）
```

### 6.4 月分析流程

```
每月1日 09:00 汇总上月数据
  ↓
每月1日 10:00 计算目标完成率
  ↓
每月1日 11:00 更新看板
  ↓
每月1日 14:00 生成月报
  ↓
每月1日 15:00 策略复盘与调整
```

## 七、自动化建议

### 7.1 可自动采集的数据

| 平台 | 数据项 | 自动化方案 | 难度 | 优先级 |
|------|--------|------------|------|--------|
| 公众号 | 粉丝数、文章数据 | 公众号 API + 定时脚本 | 中 | P1 |
| CSDN | 浏览量、粉丝数 | 爬虫 + 定时任务 | 低 | P2 |
| 小红书 | 浏览量、互动数 | 爬虫（有风险） | 高 | P3 |
| 抖音 | 播放量、互动数 | 创作者开放平台 API | 中 | P2 |
| 知识付费 | 订单、收入 | 销售平台 API | 中 | P1 |

### 7.2 自动化实现路径

#### 阶段一：手动采集（当前）

- 每日固定时间手动登录各平台采集
- 使用模板记录，保证格式统一
- 耗时：约 30 分钟/天

#### 阶段二：半自动采集（1-2个月后）

- 开发数据采集脚本
- 自动抓取 CSDN、公众号等可 API 访问的平台
- 小红书、抖音等仍需手动
- 耗时：约 15 分钟/天

#### 阶段三：全自动采集（3个月后）

- 所有平台接入自动化采集
- 数据自动入库、看板自动更新
- 异常自动预警
- 耗时：约 5 分钟/天（仅审核）

### 7.3 必须手动记录的数据

| 数据项 | 原因 | 建议 |
|--------|------|------|
| 内容发布时间 | 需人工确认发布成功 | 发布时即时记录 |
| 内容标题/描述 | 非结构化数据 | 发布时即时记录 |
| 竞品数据 | 需人工分析判断 | 每周固定时间采集 |
| 用户反馈 | 定性数据 | 收集后整理 |
| 行业热点 | 需人工判断相关性 | 随时记录 |

## 八、数据质量保障

### 8.1 数据校验规则

| 规则 | 说明 | 处理方式 |
|------|------|----------|
| 完整性 | 所有必填字段均有值 | 缺失字段标记为 N/A |
| 一致性 | 同一数据在不同来源一致 | 以官方后台数据为准 |
| 合理性 | 数据在合理范围内 | 异常数据标记并核实 |
| 时效性 | 数据在规定时间内采集 | 超时数据标注采集时间 |

### 8.2 异常处理流程

```
发现异常数据
  ↓
标记异常（红色高亮）
  ↓
记录异常原因
  ↓
二次核实（重新采集）
  ↓
确认后更新
```

### 8.3 数据备份

- **每日备份**：日数据文件自动同步到 Git
- **每周备份**：周汇总数据归档
- **每月备份**：月度数据导出为 CSV 存档

## 十、常见问题

### Q1：某平台数据异常怎么办？

1. 先确认是否为平台数据延迟（等待 1-2 小时后重试）
2. 检查是否有内容被下架或限流
3. 对比其他平台数据，判断是否为行业性问题
4. 记录异常并持续观察

### Q2：忘记采集怎么办？

1. 当日补采，但需标注「补采」
2. 如果超过 3 天，尽量采集最近 7 天的汇总数据
3. 设置手机提醒，避免遗忘

### Q3：多个账号如何处理？

1. 每个账号独立记录
2. 汇总时合并计算
3. 在看板中可分开展示或合并展示
