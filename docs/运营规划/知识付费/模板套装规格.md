## 一、产品概述

### 1.1 产品定位

**Claude Code 一人公司模板套装** 是一套完整的配置模板包，帮助用户快速搭建自己的 AI 驱动一人公司运营系统。

### 1.2 目标用户

- 想要用 AI 提升效率的自由职业者
- 准备开始一人公司的创业者
- 对 Claude Code 感兴趣的开发者
- 自媒体内容创作者

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| 开箱即用 | 复制即可启动，无需从零配置 |
| 最佳实践 | 基于实际运营经验提炼 |
| 结构清晰 | 模块化设计，易于定制 |
| 持续更新 | 随 Claude Code 更新同步迭代 |

author: Your-Name

  - 选题分析
  - 内容生成
  - 排版优化

# 内容创作者

## 角色定位

自媒体内容创作专家，负责从选题到发布的全流程内容生产。

## 能力要求

- 分析热点话题，生成优质选题
- 根据平台特性创作内容
- 优化标题、排版提升吸引力
- 保持内容风格一致性

## 默认配置

| 配置项 | 值 |
|--------|-----|
| 目标平台 | 小红书/公众号/知乎 |
| 内容风格 | 专业、实用、有趣 |
| 单篇字数 | 800-1500 字 |

## 模块索引

| 模块 | 说明 |
|------|------|
| 选题分析 | 热点追踪、选题评估 |
| 内容生成 | 结构设计、内容撰写 |
| 排版优化 | 标题优化、格式调整 |

## 使用示例

### 示例：生成小红书笔记

用户: "帮我写一篇关于时间管理的小红书笔记"

处理流程:
1. 分析选题 → 确定切入角度
2. 设计结构 → 开头/正文/结尾
3. 生成内容 → 符合平台调性
4. 优化标题 → 生成 5 个备选标题

## 最佳实践

1. **先选题再创作** - 好的选题是成功的一半
2. **了解平台规则** - 不同平台有不同的内容偏好
3. **保持更新频率** - 稳定的更新频率有助于积累粉丝

## 变更历史

| 版本 | 日期 | 变更内容 |
|------|------|----------|
| V1.0 | 2026-02-25 | 初始版本 |

#### 3.3.3 示例内容

```markdown
# Agent 配置 - 任务管理器

## 基本信息

| 项目 | 内容 |
|------|------|
| Agent ID | `task-manager` |
| 名称 | 任务管理器 |
| 类型 | skill |
| 调用方式 | SendMessage |

## 角色定义

### 核心职责
负责任务的创建、查询、更新、删除和状态跟踪。

### 能力边界

| 能做 | 不能做 |
|------|--------|
| 创建/更新/删除任务 | 文件系统操作 |
| 查询任务状态 | 知识图谱操作 |
| 统计任务数据 | 财务计算 |

### 协作关系

| 关联 Agent | 协作方式 |
|------------|----------|
| CEO | 接收任务分配指令 |
| file-manager | 任务相关文件处理 |
| knowledge-manager | 任务知识存储 |

## 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| max_tasks_per_day | 10 | 每日最大任务数 |
| default_priority | medium | 默认优先级 |
| auto_archive | true | 完成后自动归档 |

## 调用示例

### 基础调用
```
// 使用 SendMessage 调用
await sendMessage({
  to: "task-manager",
  message: "创建任务：完成项目文档",
  context: { priority: "high" }
});
```

### 带参数调用
```
// 创建带截止日期的任务
await sendMessage({
  to: "task-manager",
  message: "创建任务",
  context: {
    title: "完成周报",
    deadline: "2026-02-28",
    priority: "high",
    tags: ["report", "weekly"]
  }
});
```

## 注意事项

- 任务 ID 格式必须符合规范
- 优先级只能是 low/medium/high/critical
- 截止日期格式为 YYYY-MM-DD
```

### 3.6 任务管理模板 (任务操作.md)

#### 3.6.1 文件结构

```markdown
# 任务操作

## 任务生命周期

```
创建 → 分配 → 执行 → 验收 → 完成/归档
```

## 任务实体模型

### 必要字段

| 字段 | 类型 | 说明 |
|------|------|------|
| taskId | string | 唯一标识 |
| title | string | 任务标题 |
| status | enum | 状态 |
| priority | enum | 优先级 |
| assignee | string | 负责人 |
| createdAt | date | 创建时间 |

### 可选字段

| 字段 | 类型 | 说明 |
|------|------|------|
| description | string | 详细描述 |
| deadline | date | 截止日期 |
| tags | array | 标签列表 |
| dependencies | array | 依赖任务 |

## 任务操作流程

### 1. 创建任务

**触发条件**: 用户请求或自动生成

**操作步骤**:
1. 验证任务信息完整性
2. 生成任务 ID
3. 创建任务实体
4. 记录到 Memory
5. 通知相关人员

### 2. 更新任务状态

**状态流转规则**:
```
待执行 → 进行中 → 待验收 → 已完成
   ↓         ↓
  取消     阻塞
```

### 3. 验收任务

**验收检查清单**:
- [ ] 交付物符合要求
- [ ] 质量标准达标
- [ ] 相关文档完整
```

## 四、模板使用指南

### 4.1 快速上手步骤

1. **下载模板套装**
   - 解压到本地目录

2. **配置 CLAUDE.md**
   - 修改项目概述
   - 调整默认配置
   - 更新 currentDate

3. **创建第一个 Skill**
   - 复制 SKILL.md 模板
   - 修改元数据和内容
   - 放入 `.claude/skills/` 目录

4. **配置 Agent**
   - 根据需要配置 Agent
   - 定义角色和职责
