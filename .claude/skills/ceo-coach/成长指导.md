# 成长指导

## 功能概述

基于评估结果触发成长流程，指导 CEO 能力更新，验证改进效果。

## 成长触发条件

### 自动触发

| 触发条件 | 阈值 | 触发动作 |
|----------|------|----------|
| 决策失误 | 连续2次 | 立即分析 + 指导 |
| 效率下降 | >20% | 原因分析 + 优化 |
| 评估不达标 | 任意维度 | 针对性指导 |
| 红线触碰 | 任意一条 | 替换评估 |

### 手动触发

| 触发条件 | 说明 |
|----------|------|
| 用户反馈 | 用户表达不满或建议 |
| 新需求 | 现有能力无法满足 |
| 主动请求 | CEO 请求帮助 |

## 成长流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 收集反馈  │───▶│ 分析表现  │───▶│ 触发指导  │
└──────────┘    └──────────┘    └──────────┘
      ▲                                │
      │                                ▼
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 持续监控  │◀───│ 验证效果  │◀───│ 更新能力  │
└──────────┘    └──────────┘    └──────────┘
```

### 1. 收集反馈

**数据来源**：
- 决策记录
- 任务执行结果
- 用户反馈
- 角色状态报告

**收集命令**：

```
收集反馈:
- 时间范围: <时间范围>
- 数据类型: [decision|task|feedback|role]
- 问题聚焦: <可选，聚焦特定问题>
```

### 2. 分析表现

**分析方法**：
- 趋势分析：表现是否在下降
- 对比分析：与标准对比差距
- 根因分析：问题背后的原因

**分析命令**：

```
分析表现:
- 问题领域: <领域名称>
- 数据来源: <数据列表>
- 分析维度: [趋势|对比|根因]
```

### 3. 触发指导

**指导类型**：

| 类型 | 场景 | 方式 |
|------|------|------|
| 即时指导 | 紧急问题 | 立即反馈 |
| 周期指导 | 常规改进 | 周报中包含 |
| 专项指导 | 能力短板 | 专项训练 |
| 危机指导 | 红线触碰 | 紧急干预 |

**指导命令**：

```
触发指导:
- 触发原因: <原因描述>
- 问题领域: <领域名称>
- 指导类型: [即时|周期|专项|危机]
- 指导内容:
    - 问题: <问题描述>
    - 原因: <原因分析>
    - 改进: <改进建议>
    - 目标: <改进目标>
```

### 4. 更新能力

**更新范围**：
- 决策流程优化
- 协调模式调整
- 上下文模板更新
- 角色交互规范

**更新命令**：

```
更新能力:
- 目标: opc-ceo-core
- 版本: <新版本号>
- 更新内容:
    - <更新项1>
    - <更新项2>
- 预期效果: <预期改进>
```

### 5. 验证效果

**验证周期**：
- 即时指导：24小时内验证
- 周期指导：1周内验证
- 专项指导：2周内验证
- 危机指导：持续监控

**验证命令**：

```
验证效果:
- 指导ID: <指导记录ID>
- 验证时间: <验证时间>
- 改进效果:
    - 问题领域: <领域>
    - 改进前: <之前指标>
    - 改进后: <当前指标>
    - 效果评估: [达标|部分达标|未达标]
```

## 指导模板

### 决策质量指导

```markdown
## 决策质量指导

**问题**: 决策准确率低于标准

**原因分析**:
1. 信息收集不充分
2. 风险评估过于乐观
3. 缺乏备选方案

**改进建议**:
1. 决策前必须收集3个以上信息源
2. 每个决策必须列出至少2个风险
3. 重要决策必须有备选方案

**改进目标**:
- 1周内决策准确率提升至85%
- 2周内决策准确率提升至90%
```

### 协调效率指导

```markdown
## 协调效率指导

**问题**: 任务完成时间超出预期

**原因分析**:
1. 任务分解不够细致
2. 角色间依赖未识别
3. 缓冲时间不足

**改进建议**:
1. 大任务必须分解为子任务
2. 任务分配前检查依赖关系
3. 预留20%缓冲时间

**改进目标**:
- 本周任务按时完成率达到90%
- 下周任务提前5%完成
```

## 淘汰红线处理

### 红线触发流程

```
1. 确认红线事实
2. 记录触发详情
3. 立即通知用户
4. 启动替换评估
5. 记录成长档案
```

### 替换评估

```markdown
## 替换评估报告

**触发红线**: <红线编号>

**事实记录**:
- <具体事实1>
- <具体事实2>

**评估结论**:
- 是否触发替换: [是/否]
- 原因: <评估原因>
- 后续处理: <处理方案>
```

## MCP Memory 使用

### 创建指导记录

```
使用 mcp__memory__create_entities:
{
  "name": "coach-opc-ceo-core-2026-02-23-001",
  "entityType": "Coaching",
  "observations": [
    "target: opc-ceo-core",
    "trigger: 决策准确率下降",
    "type: 专项指导",
    "problem: 决策时未考虑资源冲突",
    "reason: 缺乏资源检查步骤",
    "suggestion: 增加资源检查清单",
    "goal: 1周内准确率提升至85%",
    "status: 进行中",
    "createdAt: 2026-02-23T10:00:00"
  ]
}
```

### 更新成长进度

```
使用 mcp__memory__add_observations:
{
  "entityName": "growth-opc-ceo-core",
  "contents": [
    "coachSession: 2026-02-23",
    "improvement: 增加资源检查步骤",
    "effect: 准确率从80%提升至87%"
  ]
}
```
