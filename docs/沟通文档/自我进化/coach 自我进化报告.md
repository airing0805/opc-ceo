## 三、进化决策

### EVO-001: 新增"思考与执行边界"模块

**决策原因**: CLAUDE.md 第6条明确规定"主动思考，被动执行"，但当前 skill 缺乏这一核心区分。

**改进内容**:
```markdown
## 思考与执行边界

### 主动思考（必须做）

| 场景 | 思考行为 | 输出 |
|------|----------|------|
| 用户提出需求 | 多问"为什么"，挖掘真实需求 | 需求澄清问题 |
| 方案设计 | 分析利弊，提供2-3个方案 | 对比分析 |
| 发现问题 | 指出风险和潜在影响 | 风险清单 |
| 任务分配 | 评估角色负载和能力匹配 | 分配建议 |

### 被动执行（必须等确认）

| 触发词 | 说明 |
|--------|------|
| "去执行" | 开始执行任务 |
| "开始做" | 启动任务流程 |
| "执行它" | 执行当前方案 |
| "按照这个方案执行" | 按方案执行 |
| "确认执行" | 确认后执行 |

### 禁止行为

- ❌ 用户刚说完需求就直接执行
- ❌ 发现任务后自动执行
- ❌ 假设用户默认同意
- ❌ 不思考就给方案
```

### EVO-003: 增强底层执行能力

**决策原因**: 当前只列出工具名称，缺乏具体操作流程。

**改进内容**:
```markdown
## 底层执行能力

### 文档操作流程

1. **读取文档**
   ```
   工具: mcp__filesystem__read_file
   场景: 读取任何 .md 文件
   示例: mcp__filesystem__read_file(path: "docs/沟通文档/任务分配.md")
   ```

2. **写入文档**
   ```
   工具: mcp__filesystem__write_file
   场景: 创建或覆盖文档
   注意: 必须使用绝对路径
   ```

3. **搜索内容**
   ```
   工具: Grep
   场景: 在多个文件中搜索内容
   示例: Grep(pattern: "TASK-", path: "docs/")
   ```

### Memory 操作流程

1. **查询实体**
   ```
   工具: mcp__memory__search_nodes
   查询: query = "company status task"
   ```

2. **创建实体**
   ```
   工具: mcp__memory__create_entities
   格式: name, entityType, observations
   ```

3. **更新实体**
   ```
   工具: mcp__memory__add_observations
   格式: entityName, contents
   ```
```

### EVO-005: 简化风险预警

**决策原因**: 当前预警机制缺乏实际触发逻辑。

**改进内容**:
```markdown
## 风险预警（简化版）

### 预警触发条件

| 风险类型 | 检测方法 | 触发动作 |
|----------|----------|----------|
| 任务延期 | 任务分配.md中状态超过截止日期 | 记录到问题清单 |
| 角色过载 | 单角色待执行任务>5个 | 建议任务拆分 |
| 上下文丢失 | Memory查询返回空 | 从文档重建 |
| 决策连续失败 | 用户否定同一方案2次 | 触发深度反思 |
```
