# 表现评估

## 动态执行规范

**评估前必须**：
1. 从 Memory 读取相关实体（决策记录、任务状态等）
2. 从 Memory 读取当前标准
3. 读取沟通文档了解上下文
4. 再进行评估

---

## 评估维度

### 1. 决策审查

**数据来源**：Memory 中的 Decision 实体

**评估内容**：
- 决策是否有充分依据
- 决策结果是否符合预期
- 是否有遗漏的风险因素

**评估输出**：
```
决策评估:
- 时间范围: 从 Memory 读取
- 决策数量: 统计得出
- 正确率: 计算得出
- 问题决策: 列出 ID 和原因
```

### 2. 效率评估

**数据来源**：Memory 中的 Task 实体

**评估内容**：
- 任务是否按时完成
- 是否有提前完成
- 延期原因分析

**评估输出**：
```
效率评估:
- 任务总数: 统计得出
- 按时完成: 统计得出
- 提前完成: 统计得出
- 延期完成: 统计得出
- 效率比率: 计算得出
```

### 3. 上下文检查

**数据来源**：Memory 中的 Communication 实体

**评估内容**：
- 上下文是否完整
- 信息传递是否准确
- 是否有信息丢失

**评估输出**：
```
上下文评估:
- 会话数: 统计得出
- 完整记录: 统计得出
- 部分缺失: 统计得出
- 信息失真: 统计得出
```

### 4. 角色状态

**数据来源**：Memory 中各角色的运行记录

**评估内容**：
- 各角色运转是否正常
- 是否有角色被闲置或过载
- 角色间协调是否顺畅

**评估输出**：
```
角色状态:
- 活跃角色: 从 Memory 读取
- 问题角色: 识别列出
- 协调问题: 识别列出
```

## 评估周期

| 类型 | 频率 | 触发方式 |
|------|------|----------|
| 周评估 | 每周 | 定期/手动 |
| 月评估 | 每月 | 定期/手动 |
| 阶段评估 | 阶段末 | 阶段完成时 |

## Memory 使用

### 创建评估记录

```
mcp__memory__create_entities:
{
  "entities": [{
    "name": "eval-YYYY-MM-DD-NNN",
    "entityType": "Evaluation",
    "observations": [
      "period: YYYY-MM-DD 至 YYYY-MM-DD",
      "type: weekly/monthly/stage",
      "decisionQuality: <值>",
      "coordinationEfficiency: <值>",
      "contextManagement: <值>",
      "reportQuality: <值>",
      "roleManagement: <值>",
      "overallLevel: 合格/优秀/卓越",
      "issues: [<问题列表>]",
      "recommendations: [<建议列表>]",
      "createdAt: YYYY-MM-DDTHH:MM:SS"
    ]
  }]
}
```

### 查询历史评估

```
mcp__memory__search_nodes:
query: "Evaluation weekly"
```
