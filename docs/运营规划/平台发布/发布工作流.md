## 一、工作流概述

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| CEO主导 | 内容准备、格式适配、素材整理由CEO完成 |
| 用户轻操作 | 用户只需复制粘贴，无需内容创作 |
| 流程标准化 | 统一的发布流程，减少操作失误 |
| 可追踪记录 | 每次发布都有完整记录 |

### 1.2 角色分工

| 角色 | 职责 | 参与度 |
|------|------|--------|
| **CEO** | 内容生产、格式适配、素材准备、记录更新 | 70% |
| **用户** | 平台登录、复制发布、链接反馈 | 30% |

### 1.3 工作流总览

```
┌─────────────────────────────────────────────────────────────┐
│                      CEO 执行阶段                            │
├─────────────────────────────────────────────────────────────┤
│  内容生产 → 格式适配 → 素材准备 → 发布清单 → 交付用户       │
└────────────────────────────┬────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────┐
│                      用户操作阶段                            │
├─────────────────────────────────────────────────────────────┤
│  账号登录 → 复制内容 → 上传配图 → 调整格式 → 确认发布       │
└────────────────────────────┬────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────┐
│                      协作完成阶段                            │
├─────────────────────────────────────────────────────────────┤
│  链接反馈 → 更新追踪表 → 数据采集 → 效果分析                │
└─────────────────────────────────────────────────────────────┘
```

## 二、发布前准备清单

### 2.1 CEO准备项

| 序号 | 检查项 | 状态 | 备注 |
|------|--------|------|------|
| 1 | 文章内容已生成 | ☐ | Markdown格式 |
| 2 | 标题已优化（含关键词） | ☐ | 符合平台规范 |
| 3 | 摘要/引言已完成 | ☐ | 50-100字 |
| 4 | 封面图已准备 | ☐ | 1242x675px |
| 5 | 配图已准备（3-5张） | ☐ | 压缩优化 |
| 6 | 标签已选定（3-5个） | ☐ | 热门+精准 |
| 7 | 平台格式已适配 | ☐ | 按平台规范 |
| 8 | 发布清单已生成 | ☐ | 包含所有素材 |

### 2.2 用户准备项

| 序号 | 检查项 | 状态 | 备注 |
|------|--------|------|------|
| 1 | 账号已登录 | ☐ | 各平台均可访问 |
| 2 | 发布后台已打开 | ☐ | 准备好编辑页面 |
| 3 | 时间充裕 | ☐ | 每篇约10分钟 |
| 4 | 网络稳定 | ☐ | 避免中断 |

## 三、各平台发布步骤

### 3.1 知乎发布流程

**入口**：https://www.zhihu.com/creator/content-management/all

```
步骤1：创建内容
├── 点击「写文章」或「发布想法」
└── 进入编辑器

步骤2：填写标题
├── 复制CEO提供的标题
├── 检查字数（不超过30字）
└── 确保无敏感词

步骤3：粘贴正文
├── 复制Markdown内容
├── 粘贴到编辑器
├── 检查格式是否正确
└── 调整段落间距

步骤4：上传配图
├── 点击图片位置
├── 上传CEO准备的图片
└── 检查显示效果

步骤5：设置选项
├── 选择话题标签（3-5个）
├── 设置封面图（可选）
├── 开启/关闭评论
└── 设置发布时间（可选）

步骤6：预览与发布
├── 点击「预览」检查效果
├── 确认无误后点击「发布」
└── 复制文章链接

步骤7：反馈记录
├── 将链接发送给CEO
├── 记录发布时间
└── 更新发布追踪表
```

**注意事项**：
- 知乎对营销敏感，避免过多外链
- 首段要吸引人，影响完读率
- 适当互动回复，提升权重

---

### 3.2 简书发布流程

**入口**：https://www.jianshu.com/writer#/

```
步骤1：新建文章
├── 点击「写文章」
└── 进入Markdown编辑器

步骤2：填写标题
├── 复制CEO提供的标题
└── 检查字数（建议20-30字）

步骤3：粘贴内容
├── 复制Markdown内容
├── 直接粘贴（简书原生支持Markdown）
└── 预览检查格式

步骤4：上传配图
├── 拖拽图片到编辑器
├── 或使用图床链接
└── 检查图片显示

步骤5：专题投稿（重要）
├── 点击「投稿」
├── 选择相关专题（最多5个）
├── 优先选择关注人数多的专题
└── 等待收录

步骤6：发布文章
├── 点击「发布」
└── 复制文章链接

步骤7：反馈记录
├── 将链接发送给CEO
└── 更新发布追踪表
```

**注意事项**：
- 简书专题投稿很重要，影响曝光
- 代码块必须标注语言
- 图片宽度不超过600px

### 3.3 CSDN发布流程

**入口**：https://mp.csdn.net/mp_blog/creation/editor

```
步骤1：新建博客
├── 点击「写博客」
└── 进入编辑器（可选Markdown模式）

步骤2：填写标题
├── 复制CEO提供的标题
├── 检查关键词位置
└── 添加年份（如"2026最新"）

步骤3：粘贴内容
├── 切换到Markdown模式
├── 复制Markdown内容
├── 粘贴到编辑器
└── 添加目录 @[toc]

步骤4：上传配图
├── 拖拽图片上传
├── CSDN自动存储
└── 检查图片清晰度

步骤5：设置分类标签
├── 选择文章分类
├── 添加标签（3-5个）
├── 选择专栏（如有）
└── 设置文章类型

步骤6：封面设置
├── 上传封面图（必需）
├── 选择16:9比例
└── 检查封面效果

步骤7：摘要填写
├── 填写文章摘要
├── 包含核心关键词
└── 50-100字

步骤8：发布设置
├── 选择「立即发布」或「定时发布」
├── 选择「原创」或「转载」
├── 点击「发布文章」
└── 复制文章链接

步骤9：反馈记录
├── 将链接发送给CEO
└── 更新发布追踪表
```

**注意事项**：
- 封面图是必需的，要提前准备
- 标题SEO优化很重要
- 原创标签有利于推荐

### 3.4 掘金发布流程

**入口**：https://juejin.cn/creator/content

```
步骤1：创建文章
├── 点击「写文章」
└── 进入Markdown编辑器

步骤2：填写标题
├── 复制CEO提供的标题
└── 检查标题规范

步骤3：粘贴内容
├── 复制Markdown内容
├── 粘贴到编辑器
├── 预览检查格式
└── 检查代码高亮

步骤4：上传配图
├── 拖拽或粘贴图片
├── 掘金自动存储
└── 检查图片显示

步骤5：设置分类
├── 选择文章分类
├── 添加标签
├── 选择话题（如有）
└── 设置可见范围

步骤6：封面设置
├── 上传封面图（可选）
└── 选择推荐比例

步骤7：发布
├── 点击「发布」
├── 等待审核（如有）
└── 复制文章链接

步骤8：反馈记录
├── 将链接发送给CEO
└── 更新发布追踪表
```

**注意事项**：
- 掘金代码高亮效果最好
- 技术内容更受欢迎
- 沸点互动可增加曝光

## 四、批量发布流程

### 4.1 单日发布流程（2-3篇）

```
09:00 - 09:30 │ 准备阶段
              │ ├── CEO发送当日发布清单
              │ ├── 用户确认发布计划
              │ └── 登录各平台账号

09:30 - 10:30 │ 第一批发布（2篇）
              │ ├── 发布CSDN文章1
              │ ├── 发布知乎文章1
              │ └── 反馈链接

14:00 - 15:00 │ 第二批发布（1篇）
              │ ├── 发布简书文章1
              │ └── 反馈链接

15:00 - 15:30 │ 记录更新
              │ ├── 更新发布追踪表
              │ └── 确认所有链接已记录
```

### 4.2 周度发布流程

| 日期 | 任务 | 发布量 | 平台 |
|------|------|--------|------|
| 周一 | 首批发布 | 2-3篇 | CSDN + 知乎 |
| 周二 | 持续发布 | 2-3篇 | 简书 + 掘金 |
| 周三 | 持续发布 | 2-3篇 | CSDN + 知乎 |
| 周四 | 持续发布 | 2-3篇 | 简书 + 掘金 |
| 周五 | 持续发布 | 2-3篇 | CSDN + 知乎 |
| 周六 | 数据采集 | - | 全平台 |
| 周日 | 复盘优化 | - | - |

## 五、发布后验证流程

### 5.1 即时验证（发布后5分钟）

| 检查项 | 操作 | 状态 |
|--------|------|------|
| 文章可访问 | 点击链接确认 | ☐ |
| 格式正确 | 检查排版显示 | ☐ |
| 图片显示 | 确认所有图片加载 | ☐ |
| 标签生效 | 检查标签是否显示 | ☐ |
| 链接记录 | 更新追踪表 | ☐ |

### 5.2 短期验证（发布后24小时）

| 检查项 | 操作 | 状态 |
|--------|------|------|
| 搜索收录 | 搜索标题确认收录 | ☐ |
| 推荐情况 | 查看是否被推荐 | ☐ |
| 初始数据 | 记录初始阅读/点赞 | ☐ |
| 评论区 | 检查是否需要回复 | ☐ |

### 5.3 中期验证（发布后7天）

| 检查项 | 操作 | 状态 |
|--------|------|------|
| 数据采集 | 采集7日数据 | ☐ |
| 效果分析 | 分析阅读/互动数据 | ☐ |
| 优化建议 | 总结优化方向 | ☐ |
| 追踪表更新 | 更新最终数据 | ☐ |

## 六、常见问题处理

### 6.1 发布失败

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 内容审核不通过 | 敏感词/违规内容 | 修改后重新提交 |
| 图片上传失败 | 格式/大小问题 | 压缩或转换格式 |
| 登录态失效 | cookie过期 | 重新登录 |
| 网络超时 | 网络问题 | 稍后重试 |

### 6.2 格式问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 代码不高亮 | 未标注语言 | 添加语言标识 |
| 图片不显示 | 链接失效 | 重新上传 |
| 排版混乱 | Markdown不兼容 | 手动调整 |
| 表格错位 | 格式问题 | 检查表格语法 |

### 6.3 效果问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 无阅读量 | 标题不吸引 | 优化标题 |
| 无互动 | 内容质量低 | 提升内容质量 |
| 无收录 | SEO问题 | 优化关键词 |
| 被限流 | 营销过度 | 减少营销内容 |
