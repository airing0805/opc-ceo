## 一、方案说明

本文档定义各P0平台（知乎、简书、CSDN、掘金）的首发内容分配策略，确保内容与平台调性匹配，实现最佳发布效果。

### 分配原则

1. **平台调性匹配** - 根据各平台用户偏好分配内容类型
2. **内容差异化** - 同一选题在不同平台采用不同标题和角度
3. **首发+重发结合** - 首发新内容+历史内容差异化重发
4. **数据驱动优化** - 根据数据反馈调整分配策略

## 四、内容适配指南

### 4.1 知乎适配规则

| 要素 | 适配建议 |
|------|----------|
| 标题 | 使用疑问句、引发思考型标题 |
| 开头 | 抛出痛点问题，引起共鸣 |
| 结构 | 总-分-总，结论先行 |
| 深度 | 提供独家见解和深度分析 |
| 结尾 | 总结要点，引导评论互动 |
| 标签 | 添加相关话题标签 |

### 4.2 简书适配规则

| 要素 | 适配建议 |
|------|----------|
| 标题 | 情感化、故事性强 |
| 开头 | 以故事或个人经历引入 |
| 结构 | 轻松自然，段落短小 |
| 风格 | 真诚、有温度 |
| 结尾 | 引导点赞和关注 |
| 配图 | 高清美图为主 |

### 4.3 CSDN适配规则

| 要素 | 适配建议 |
|------|----------|
| 标题 | 包含技术关键词，便于搜索 |
| 开头 | 直接说明本文解决的问题 |
| 结构 | 步骤清晰，代码完整 |
| 代码 | 必须有可运行的代码示例 |
| 结尾 | 提供参考资料和延伸阅读 |
| SEO | 标题包含热门关键词 |

### 4.4 掘金适配规则

| 要素 | 适配建议 |
|------|----------|
| 标题 | 简洁有力，突出干货 |
| 开头 | 说明读者能获得什么 |
| 结构 | 要点式，结论先行 |
| 代码 | 短代码片段，关键部分高亮 |
| 结尾 | 引导点赞、在看 |
| 互动 | 参与沸点讨论 |

## 五、内容分发流程

### 5.1 标准发布流程

```
选题确认 → 内容创作 → 平台适配 → 定时发布 → 数据追踪 → 优化迭代
```

### 5.2 差异化策略

| 环节 | 操作 | 说明 |
|------|------|------|
| 选题 | 统一选题库 | 所有平台共用选题库 |
| 创作 | 一稿多版 | 根据平台调性调整 |
| 标题 | 平台适配 | 每个平台不同标题 |
| 封面 | 统一风格 | 品牌一致性 |
| 发布时间 | 平台最佳时间 | 详见发布时刻表 |
| 互动 | 平台特色互动 | 评论回复、沸点互动 |

### 5.3 发布时间表
