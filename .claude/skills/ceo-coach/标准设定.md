# 标准设定

## 功能概述

定义 CEO 各能力维度的量化标准，设定成长时间窗口和阶段要求。

## 能力维度

### 1. 决策质量

衡量 CEO 做出正确决策的比例。

| 等级 | 标准 | 定义 |
|------|------|------|
| 合格 | 80% | 5次决策中4次正确 |
| 优秀 | 90% | 10次决策中9次正确 |
| 卓越 | 95%+ | 几乎不出错，且有预判 |

**评估方法**：
- 记录每次决策及结果
- 统计正确决策占比
- 定期复盘错误决策

### 2. 协调效率

衡量任务协调和完成的时间效率。

| 等级 | 标准 | 定义 |
|------|------|------|
| 合格 | 按时完成 | 在预期时间内完成 |
| 优秀 | 提前10% | 比预期快10%完成 |
| 卓越 | 提前20%+ | 显著快于预期 |

**评估方法**：
- 记录任务预期时间
- 记录实际完成时间
- 计算效率比率

### 3. 上下文管理

衡量信息管理和传递的质量。

| 等级 | 标准 | 定义 |
|------|------|------|
| 合格 | 信息完整 | 不遗漏关键信息 |
| 优秀 | 信息精准 | 信息准确且切中要点 |
| 卓越 | 信息预判 | 提前准备可能需要的信息 |

**评估方法**：
- 检查上下文记录完整性
- 评估信息传递准确性
- 观察预判性准备

### 4. 结果汇报

衡量向用户汇报的质量。

| 等级 | 标准 | 定义 |
|------|------|------|
| 合格 | 清晰全面 | 用户能理解全部内容 |
| 优秀 | 简洁有力 | 核心突出，细节可查 |
| 卓越 | 可直接汇报 | 格式规范，无需修改 |

**评估方法**：
- 用户反馈
- 汇报内容结构化程度
- 是否需要二次整理

### 5. 角色管理

衡量对各专业角色的管理能力。

| 等级 | 标准 | 定义 |
|------|------|------|
| 合格 | 正常运转 | 各角色按时完成任务 |
| 优秀 | 持续优化 | 主动发现问题并优化 |
| 卓越 | 主动进化 | 预判需求，提前进化 |

**评估方法**：
- 各角色任务完成率
- 主动优化次数
- 预判性能力更新

## 时间窗口

### 基础期（第1-2周）

**目标**：掌握所有核心能力

**要求**：
- 理解所有8个下属角色的职责
- 能够正确分发任务到对应角色
- 建立基本的上下文管理习惯
- 完成首次决策记录

**不达标后果**：警告 + 额外指导

### 成长期（第3-4周）

**目标**：达到「优秀」水平

**要求**：
- 决策准确率 ≥ 90%
- 任务提前 10% 完成
- 上下文精准传递
- 汇报简洁有力
- 触发至少1次角色优化

**不达标后果**：留用观察 + 强化指导

### 成熟期（第5-12周）

**目标**：达到「卓越」水平

**要求**：
- 决策准确率 ≥ 95%
- 任务提前 20% 完成
- 信息预判性准备
- 汇报可直接使用
- 主动进化角色能力

**不达标后果**：重新评估是否适合

## 标准更新

### 更新触发条件

| 触发条件 | 说明 |
|----------|------|
| 业务变化 | 公司业务方向调整 |
| 技术升级 | 新技术或工具引入 |
| 用户反馈 | 用户期望提升 |
| 行业标杆 | 行业最佳实践更新 |

### 更新流程

```
1. 识别更新需求
2. 评估影响范围
3. 制定新标准
4. 通知 CEO
5. 实施新标准
6. 验证效果
```

## MCP Memory 使用

### 创建标准实体

```
使用 mcp__memory__create_entities:
{
  "name": "standard-opc-ceo-core",
  "entityType": "Standard",
  "observations": [
    "version: 1.0.0",
    "decisionQuality_base: 80%",
    "decisionQuality_good: 90%",
    "decisionQuality_excellent: 95%",
    "coordinationEfficiency_base: 按时",
    "coordinationEfficiency_good: 提前10%",
    "coordinationEfficiency_excellent: 提前20%",
    "updatedAt: 2026-02-23"
  ]
}
```

### 查询当前标准

```
使用 mcp__memory__open_nodes:
names: ["standard-opc-ceo-core"]
```
