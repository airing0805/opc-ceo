## 一、流程概述

### 1.1 核心目标

建立一套标准化的每日内容生产流程，实现：

- **稳定产出**：每日1-2篇高质量内容
- **高效执行**：单篇内容2-3小时内完成
- **质量保证**：通过检查点确保内容质量
- **可追溯性**：完整的执行记录

### 1.2 角色分工

| 角色 | 职责 | 参与环节 |
|------|------|----------|
| **Coach** | 制定规范、选题规划、质量标准 | 选题确认、质量审核 |
| **CEO** | 执行生产、素材准备、发布运营 | 全流程执行 |
| **用户** | 最终确认、协助操作 | 关键节点确认 |

## 二、每日时间规划

### 2.1 时间段分配

| 时间段 | 环节 | 时长 | 核心任务 |
|--------|------|------|----------|
| **早晨准备期** | | | |
| 07:00-08:00 | 选题确认 | 60分钟 | 从选题库选取当日选题，确认发布计划 |
| **上午执行期** | | | |
| 09:00-10:30 | 资料收集 | 90分钟 | 收集素材、参考资料、案例 |
| 10:30-12:00 | 内容写作 | 90分钟 | 撰写初稿，完成核心内容 |
| **下午完善期** | | | |
| 14:00-15:00 | 内容审核 | 60分钟 | 自我审核，修改优化 |
| 15:00-16:00 | 配图排版 | 60分钟 | 准备配图，完成排版 |
| 16:00-17:00 | 平台发布 | 60分钟 | 多平台分发，设置标签 |
| **晚间复盘期** | | | |
| 20:00-20:30 | 数据复盘 | 30分钟 | 查看数据，记录反馈 |

### 2.2 时间规划原则

| 原则 | 说明 |
|------|------|
| **黄金时间做核心事** | 上午精力充沛时专注写作 |
| **批量处理同类事** | 集中时间处理配图、发布 |
| **预留缓冲时间** | 每个环节预留10-15分钟缓冲 |
| **固定时间复盘** | 每日晚间固定时间复盘 |

### 2.3 不同类型内容时间调整

| 内容类型 | 建议时间 | 调整说明 |
|----------|----------|----------|
| 深度干货（3000+字） | 4-5小时 | 分两天完成，第一天写作，第二天优化 |
| 工具实测（1500字） | 2-3小时 | 包含截图和测试时间 |
| 观点解读（1000字） | 1.5-2小时 | 快速响应热点 |
| 日常分享（800字） | 1-1.5小时 | 轻量内容快速产出 |

### 3.2 各步骤详细说明

#### Step 1: 选题确认

**目标**: 从选题库选取1-2篇今日要完成的内容

**输入**:
- 选题库文件：`docs/运营规划/内容规划/选题库/`
- 热点话题（如有）

**操作步骤**:

| 步骤 | 操作 | 产出 |
|------|------|------|
| 1 | 打开选题库，筛选"待执行"状态 | 待执行选题列表 |
| 2 | 评估选题优先级（热度+变现） | 排序后的选题 |
| 3 | 结合今日时间确定数量 | 今日选题（1-2篇） |
| 4 | 更新选题状态为"进行中" | 状态更新 |

**选题优先级评估表**:

| 维度 | 高（3分） | 中（2分） | 低（1分） |
|------|-----------|-----------|-----------|
| 热度 | 当前热点 | 常青话题 | 过时话题 |
| 变现 | 直接转化 | 间接转化 | 纯流量 |
| 难度 | 有现成资料 | 需要调研 | 需要深度研究 |

---

#### Step 2: 资料收集

**目标**: 收集写作所需的素材和参考资料

**资料来源**:

| 来源类型 | 适用场景 | 工具 |
|----------|----------|------|
| 搜索引擎 | 通用信息 | Google/百度 |
| 知乎/Quora | 观点案例 | 直接搜索 |
| 官方文档 | 技术内容 | 官网 |
| 数据报告 | 行业数据 | 艾瑞/易观 |
| 竞品分析 | 对比内容 | 平台搜索 |

**收集清单模板**:

```markdown
## 资料收集清单 - {选题标题}

### 核心资料
- [ ] 官方文档/公告: [链接]
- [ ] 权威数据: [数据来源]
- [ ] 专家观点: [引用]

### 案例素材
- [ ] 成功案例: [描述]
- [ ] 失败案例: [描述]
- [ ] 数据截图: [保存路径]

### 参考文章
1. [标题](链接) - 核心观点: xxx
2. [标题](链接) - 核心观点: xxx

### 待确认问题
- [ ] 问题1: xxx
- [ ] 问题2: xxx
```

#### Step 3: 内容写作

**目标**: 完成文章初稿

**写作模板选择**:

| 内容类型 | 模板文件 | 字数要求 |
|----------|----------|----------|
| 技术教程 | `内容模板/CSDN文章模板.md` | 1500-3000字 |
| 图文笔记 | `内容模板/小红书图文模板.md` | 300-800字 |
| 深度分析 | 自定义结构 | 2000-4000字 |

**写作流程**:

```
1. 确定标题（3-5个备选）
   ↓
2. 设计开头（吸引眼球）
   ↓
3. 填充核心内容（按模板结构）
   ↓
4. 添加总结和行动号召
   ↓
5. 完成初稿
```

**标题公式**:

| 类型 | 公式 | 示例 |
|------|------|------|
| 数字型 | [数字]个[方法/工具] | 10个提升效率的AI工具 |
| 对比型 | A vs B，哪个更好？ | Claude vs GPT，谁更适合程序员？ |
| 疑问型 | 为什么...？ | 为什么你的AI工具总用不好？ |
| 清单型 | [领域]必备的[数量]个[东西] | 程序员必备的5个AI助手 |

---

#### Step 4: 内容审核

**目标**: 确保内容质量达标

**审核清单**:

```markdown
## 内容审核清单

### 基础检查
- [ ] 无错别字
- [ ] 无语病
- [ ] 标点符号正确
- [ ] 格式统一

### 内容检查
- [ ] 标题吸引人（有点击欲望）
- [ ] 开头抓眼球（3秒内吸引读者）
- [ ] 结构清晰（小标题明确）
- [ ] 观点明确（有核心论点）
- [ ] 价值明确（读者能获得什么）

### 原创检查
- [ ] 查重率 < 20%
- [ ] 核心观点原创
- [ ] 引用已标注来源

### 平台适配
- [ ] 符合平台规范
- [ ] 字数符合要求
- [ ] 无敏感内容
```

#### Step 5: 配图排版

**目标**: 完成视觉优化

**配图需求**:

| 位置 | 数量 | 要求 |
|------|------|------|
| 封面图 | 1张 | 16:9，高清，吸引眼球 |
| 内容配图 | 2-5张 | 与内容相关，清晰美观 |
| 结尾图 | 0-1张 | 可选，引导关注 |

**配图来源**:

| 来源 | 占比 | 操作方式 |
|------|------|----------|
| AI生成 | 60% | 使用AI工具生成 |
| 免费图库 | 30% | Unsplash/Pexels |
| 截图 | 10% | 软件截图 |

**排版规范**:

| 元素 | 规范 |
|------|------|
| 段落 | 每段3-5行，不超过100字 |
| 小标题 | 使用H2/H3，清晰明确 |
| 重点 | 加粗/引用块突出 |
| 列表 | 使用有序/无序列表 |
| 空行 | 段落间空一行 |

### 5.2 各检查点详情

#### Q1: 选题质量检查点

| 检查项 | 标准 | 不通过处理 |
|--------|------|------------|
| 定位匹配 | 符合账号领域定位 | 重新选题 |
| 资料充足 | 有足够参考资料 | 调整选题或补充调研 |
| 价值明确 | 对读者有明确价值 | 调整角度 |
| 时效性 | 非过时话题 | 更换选题 |

#### Q2: 资料质量检查点

| 检查项 | 标准 | 不通过处理 |
|--------|------|------------|
| 来源可靠 | 官方/权威来源 | 核实或更换 |
| 数据准确 | 数据可验证 | 重新核实 |
| 观点多元 | 有多个视角 | 补充调研 |

#### Q3: 内容质量检查点

| 检查项 | 标准 | 不通过处理 |
|--------|------|------------|
| 结构完整 | 有开头/主体/结尾 | 补充完善 |
| 观点明确 | 核心观点清晰 | 重新梳理 |
| 逻辑通顺 | 段落衔接自然 | 调整顺序 |
| 字数达标 | 符合平台要求 | 增删内容 |

#### Q4: 配图质量检查点

| 检查项 | 标准 | 不通过处理 |
|--------|------|------------|
| 相关性 | 配图与内容相关 | 重新选择 |
| 清晰度 | 图片清晰无模糊 | 更换图片 |
| 版权 | 无版权问题 | 更换来源 |

#### Q5: 发布质量检查点

| 检查项 | 标准 | 不通过处理 |
|--------|------|------------|
| 格式正确 | 平台格式规范 | 重新调整 |
| 标签合适 | 3-5个相关标签 | 重新设置 |
| 链接有效 | 发布链接可访问 | 重新发布 |

#### Q6: 数据质量检查点

| 检查项 | 标准 | 不通过处理 |
|--------|------|------------|
| 数据准确 | 数据记录正确 | 重新核实 |
| 记录完整 | 所有字段填写 | 补充记录 |

## 六、常见问题处理

### 6.1 选题阶段问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 选题库没有合适选题 | 选题已用完或不符合当前阶段 | 1. 补充新选题 2. 调整选题角度 |
| 不知道选哪个选题 | 多个选题优先级相近 | 使用选题评估表打分 |
| 选题与热点冲突 | 突发热点需要响应 | 1. 插队热点选题 2. 原选题延后 |

### 6.2 写作阶段问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 写不出来 | 资料不足/思路不清 | 1. 补充资料 2. 列大纲 3. 先写易写部分 |
| 内容太长/太短 | 规划不当 | 1. 删减冗余 2. 补充案例 |
| 缺乏灵感 | 创作疲劳 | 1. 休息调整 2. 参考优秀案例 |
| 查重率高 | 原创度不够 | 1. 重新组织语言 2. 加入个人观点 |

### 6.3 配图阶段问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 找不到合适配图 | 需求不明确 | 1. 明确配图需求 2. AI生成 |
| AI生成效果差 | Prompt 不精准 | 优化 Prompt，多次尝试 |
| 版权问题 | 使用了有版权图片 | 1. 使用免费图库 2. 标注来源 |

### 6.4 发布阶段问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 发布失败 | 格式/内容问题 | 检查平台规范，调整后重试 |
| 审核不通过 | 敏感内容 | 修改敏感内容，重新提交 |
| 排版错乱 | 格式不兼容 | 使用平台编辑器调整 |

### 6.5 时间管理问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 时间不够用 | 任务过多/效率低 | 1. 减少任务量 2. 使用模板提速 |
| 经常被打断 | 外部干扰 | 设置专注时间，关闭通知 |
| 进度落后 | 预估不准 | 调整计划，使用紧急时间压缩 |
